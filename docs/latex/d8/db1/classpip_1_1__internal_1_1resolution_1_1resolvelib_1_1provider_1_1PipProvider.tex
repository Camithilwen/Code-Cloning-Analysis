    are canonicalized project names.
:params ignore_dependencies: Whether the user specified ``--no-deps``.
:params upgrade_strategy: The user-specified upgrade strategy.

 "{}Preference"{} pip.\+\_\+internal.\+resolution.\+resolvelib.\+provider.\+Pip\+Provider.\+get\+\_\+preference (\begin{DoxyParamCaption}\item[{}]{self}{, }\item[{str}]{identifier}{, }\item[{Mapping\mbox{[}str, \mbox{\hyperlink{classpip_1_1__internal_1_1resolution_1_1resolvelib_1_1base_1_1Candidate}{Candidate}}\mbox{]}}]{resolutions}{, }\item[{Mapping\mbox{[}str, Iterator\mbox{[}\mbox{\hyperlink{classpip_1_1__internal_1_1resolution_1_1resolvelib_1_1base_1_1Candidate}{Candidate}}\mbox{]}\mbox{]}}]{candidates}{, }\item[{Mapping\mbox{[}str, Iterable\mbox{[}"{}Preference\+Information"{}\mbox{]}\mbox{]}}]{information}{, }\item[{Sequence\mbox{[}"{}Preference\+Information"{}\mbox{]}}]{backtrack\+\_\+causes}{}\end{DoxyParamCaption})}

\begin{DoxyVerb}Produce a sort key for given requirement based on preference.

The lower the return value is, the more preferred this group of
arguments is.

Currently pip considers the following in order:

* Prefer if any of the known requirements is "direct", e.g. points to an
  explicit URL.
* If equal, prefer if any requirement is "pinned", i.e. contains
  operator ``===`` or ``==``.
* If equal, calculate an approximate "depth" and resolve requirements
  closer to the user-specified requirements first. If the depth cannot
  by determined (eg: due to no matching parents), it is considered
  infinite.
* Order user-specified requirements by the order they are specified.
* If equal, prefers "non-free" requirements, i.e. contains at least one
  operator, such as ``>=`` or ``<``.
* If equal, order alphabetically for consistency (helps debuggability).
\end{DoxyVerb}
