13Usage of crackfortran:
14======================
15Command line keys: -quiet,-verbose,-fix,-f77,-f90,-show,-h <pyffilename>
16 -m <module name for f77 routines>,--ignore-contains
17Functions: crackfortran, crack2fortran
18The following Fortran statements/constructions are supported
19(or will be if needed):
20 block data,byte,call,character,common,complex,contains,data,
21 dimension,double complex,double precision,end,external,function,
22 implicit,integer,intent,interface,intrinsic,
23 logical,module,optional,parameter,private,public,
24 program,real,(sequence?),subroutine,type,use,virtual,
25 include,pythonmodule
26Note: 'virtual' is mapped to 'dimension'.
27Note: 'implicit integer (z) static (z)' is 'implicit static (z)' (this is minor bug).
28Note: code after 'contains' will be ignored until its scope ends.
29Note: 'common' statement is extended: dimensions are moved to variable definitions
30Note: f2py directive: <commentchar>f2py<line> is read as <line>
31Note: pythonmodule is introduced to represent Python module
32
33Usage:
34 `postlist=crackfortran(files)`
35 `postlist` contains declaration information read from the list of files `files`.
36 `crack2fortran(postlist)` returns a fortran code to be saved to pyf-file
37
38 `postlist` has the following structure:
39 *** it is a list of dictionaries containing `blocks':
40 B = {'block','body','vars','parent_block'[,'name','prefix','args','result',
41 'implicit','externals','interfaced','common','sortvars',
42 'commonvars','note']}
43 B['block'] = 'interface' | 'function' | 'subroutine' | 'module' |
44 'program' | 'block data' | 'type' | 'pythonmodule' |
45 'abstract interface'
46 B['body'] --- list containing `subblocks' with the same structure as `blocks'
47 B['parent_block'] --- dictionary of a parent block:
48 C['body'][<index>]['parent_block'] is C
49 B['vars'] --- dictionary of variable definitions
50 B['sortvars'] --- dictionary of variable definitions sorted by dependence (independent first)
51 B['name'] --- name of the block (not if B['block']=='interface')
52 B['prefix'] --- prefix string (only if B['block']=='function')
53 B['args'] --- list of argument names if B['block']== 'function' | 'subroutine'
54 B['result'] --- name of the return value (only if B['block']=='function')
55 B['implicit'] --- dictionary {'a':<variable definition>,'b':...} | None
56 B['externals'] --- list of variables being external
57 B['interfaced'] --- list of variables being external and defined
58 B['common'] --- dictionary of common blocks (list of objects)
59 B['commonvars'] --- list of variables used in common blocks (dimensions are moved to variable definitions)
60 B['from'] --- string showing the 'parents' of the current block
61 B['use'] --- dictionary of modules used in current block:
62 {<modulename>:{['only':<0|1>],['map':{<local_name1>:<use_name1>,...}]}}
63 B['note'] --- list of LaTeX comments on the block
64 B['f2pyenhancements'] --- optional dictionary
65 {'threadsafe':'','fortranname':<name>,
66 'callstatement':<C-expr>|<multi-line block>,
67 'callprotoargument':<C-expr-list>,
68 'usercode':<multi-line block>|<list of multi-line blocks>,
69 'pymethoddef:<multi-line block>'
70 }
71 B['entry'] --- dictionary {entryname:argslist,..}
72 B['varnames'] --- list of variable names given in the order of reading the
73 Fortran code, useful for derived types.
74 B['saved_interface'] --- a string of scanned routine signature, defines explicit interface
75 *** Variable definition is a dictionary
76 D = B['vars'][<variable name>] =
77 {'typespec'[,'attrspec','kindselector','charselector','=','typename']}
78 D['typespec'] = 'byte' | 'character' | 'complex' | 'double complex' |
79 'double precision' | 'integer' | 'logical' | 'real' | 'type'
80 D['attrspec'] --- list of attributes (e.g. 'dimension(<arrayspec>)',
81 'external','intent(in|out|inout|hide|c|callback|cache|aligned4|aligned8|aligned16)',
82 'optional','required', etc)
83 K = D['kindselector'] = {['*','kind']} (only if D['typespec'] =
84 'complex' | 'integer' | 'logical' | 'real' )
85 C = D['charselector'] = {['*','len','kind','f2py_len']}
86 (only if D['typespec']=='character')
87 D['='] --- initialization expression string
88 D['typename'] --- name of the type if D['typespec']=='type'
89 D['dimension'] --- list of dimension bounds
90 D['intent'] --- list of intent specifications
91 D['depend'] --- list of variable names on which current variable depends on
92 D['check'] --- list of C-expressions; if C-expr returns zero, exception is raised
93 D['note'] --- list of LaTeX comments on the variable
94 *** Meaning of kind/char selectors (few examples):
95 D['typespec>']*K['*']
96 D['typespec'](kind=K['kind'])
97 character*C['*']
98 character(len=C['len'],kind=C['kind'], f2py_len=C['f2py_len'])
99 (see also fortran type declaration statement formats below)
100
101Fortran 90 type declaration statement format (F77 is subset of F90)
102====================================================================
103(Main source: IBM XL Fortran 5.1 Language Reference Manual)
104type declaration = <typespec> [[<attrspec>]::] <entitydecl>
105<typespec> = byte |
106 character[<charselector>] |
107 complex[<kindselector>] |
108 double complex |
109 double precision |
110 integer[<kindselector>] |
111 logical[<kindselector>] |
112 real[<kindselector>] |
113 type(<typename>)
114<charselector> = * <charlen> |
115 ([len=]<len>[,[kind=]<kind>]) |
116 (kind=<kind>[,len=<len>])
117<kindselector> = * <intlen> |
118 ([kind=]<kind>)
119<attrspec> = comma separated list of attributes.
120 Only the following attributes are used in
121 building up the interface:
122 external
123 (parameter --- affects '=' key)
124 optional
125 intent
126 Other attributes are ignored.
127<intentspec> = in | out | inout
128<arrayspec> = comma separated list of dimension bounds.
129<entitydecl> = <name> [[*<charlen>][(<arrayspec>)] | [(<arrayspec>)]*<charlen>]
130 [/<init_expr>/ | =<init_expr>] [,<entitydecl>]
131
132In addition, the following attributes are used: check,depend,note
133
134TODO:

301def openhook(filename, mode):
302 """Ensures that filename is opened with correct encoding parameter.
303
304 This function uses charset_normalizer package, when available, for
305 determining the encoding of the file to be opened. When charset_normalizer
306 is not available, the function detects only UTF encodings, otherwise, ASCII
307 encoding is used as fallback.
308 """
309 # Reads in the entire file. Robust detection of encoding.
310 # Correctly handles comments or late stage unicode characters
311 # gh-22871
312 if charset_normalizer is not None:
313 encoding = charset_normalizer.from_path(filename).best().encoding
314 else:
315 # hint: install charset_normalizer for correct encoding handling
316 # No need to read the whole file for trying with startswith
317 nbytes = min(32, os.path.getsize(filename))
318 with open(filename, 'rb') as fhandle:
319 raw = fhandle.read(nbytes)
320 if raw.startswith(codecs.BOM_UTF8):
321 encoding = 'UTF-8-SIG'
322 elif raw.startswith((codecs.BOM_UTF32_LE, codecs.BOM_UTF32_BE)):
323 encoding = 'UTF-32'
324 elif raw.startswith((codecs.BOM_LE, codecs.BOM_BE)):
325 encoding = 'UTF-16'
326 else:
327 # Fallback, without charset_normalizer
328 encoding = 'ascii'
329 return open(filename, mode, encoding=encoding)

332def is_free_format(fname):
333 """Check if file is in free format Fortran."""
334 # f90 allows both fixed and free format, assuming fixed unless
335 # signs of free format are detected.
336 result = False
337 if Path(fname).suffix.lower() in COMMON_FREE_EXTENSIONS:
338 result = True
339 with openhook(fname, 'r') as fhandle:
340 line = fhandle.readline()
341 n = 15 # the number of non-comment lines to scan for hints
342 if _has_f_header(line):
343 n = 0
344 elif _has_f90_header(line):
345 n = 0
346 result = True
347 while n > 0 and line:
348 if line[0] != '!' and line.strip():
349 n -= 1
350 if (line[0] != '\t' and _free_f90_start(line[:5])) or line[-2:-1] == '&':
351 result = True
352 break
353 line = fhandle.readline()
354 return result
