34def isclose(
35 a: Array | complex,
36 b: Array | complex,
37 *,
38 rtol: float = 1e-05,
39 atol: float = 1e-08,
40 equal_nan: bool = False,
41 xp: ModuleType | None = None,
42) -> Array:
43 """
44 Return a boolean array where two arrays are element-wise equal within a tolerance.
45
46 The tolerance values are positive, typically very small numbers. The relative
47 difference ``(rtol * abs(b))`` and the absolute difference `atol` are added together
48 to compare against the absolute difference between `a` and `b`.
49
50 NaNs are treated as equal if they are in the same place and if ``equal_nan=True``.
51 Infs are treated as equal if they are in the same place and of the same sign in both
52 arrays.
53
54 Parameters
55 ----------
56 a, b : Array | int | float | complex | bool
57 Input objects to compare. At least one must be an array.
58 rtol : array_like, optional
59 The relative tolerance parameter (see Notes).
60 atol : array_like, optional
61 The absolute tolerance parameter (see Notes).
62 equal_nan : bool, optional
63 Whether to compare NaN's as equal. If True, NaN's in `a` will be considered
64 equal to NaN's in `b` in the output array.
65 xp : array_namespace, optional
66 The standard-compatible namespace for `a` and `b`. Default: infer.
67
68 Returns
69 -------
70 Array
71 A boolean array of shape broadcasted from `a` and `b`, containing ``True`` where
72 `a` is close to `b`, and ``False`` otherwise.
73
74 Warnings
75 --------
76 The default `atol` is not appropriate for comparing numbers with magnitudes much
77 smaller than one (see notes).
78
79 See Also
80 --------
81 math.isclose : Similar function in stdlib for Python scalars.
82
83 Notes
84 -----
85 For finite values, `isclose` uses the following equation to test whether two
86 floating point values are equivalent::
87
88 absolute(a - b) <= (atol + rtol * absolute(b))
89
90 Unlike the built-in `math.isclose`,
91 the above equation is not symmetric in `a` and `b`,
92 so that ``isclose(a, b)`` might be different from ``isclose(b, a)`` in some rare
93 cases.
94
95 The default value of `atol` is not appropriate when the reference value `b` has
96 magnitude smaller than one. For example, it is unlikely that ``a = 1e-9`` and
97 ``b = 2e-9`` should be considered "close", yet ``isclose(1e-9, 2e-9)`` is ``True``
98 with default settings. Be sure to select `atol` for the use case at hand, especially
99 for defining the threshold below which a non-zero value in `a` will be considered
100 "close" to a very small or zero value in `b`.
101
102 The comparison of `a` and `b` uses standard broadcasting, which means that `a` and
103 `b` need not have the same shape in order for ``isclose(a, b)`` to evaluate to
104 ``True``.
105
106 `isclose` is not defined for non-numeric data types.
107 ``bool`` is considered a numeric data-type for this purpose.
108 """
109 xp = array_namespace(a, b) if xp is None else xp

112 return xp.isclose(a, b, rtol=rtol, atol=atol, equal_nan=equal_nan)
