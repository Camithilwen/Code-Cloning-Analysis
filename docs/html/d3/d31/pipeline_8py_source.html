83def _cached_transform(
84 sub_pipeline, *, cache, param_name, param_value, transform_params
85):
86 """Transform a parameter value using a sub-pipeline and cache the result.
87
88 Parameters
89 ----------
90 sub_pipeline : Pipeline
91 The sub-pipeline to be used for transformation.
92 cache : dict
93 The cache dictionary to store the transformed values.
94 param_name : str
95 The name of the parameter to be transformed.
96 param_value : object
97 The value of the parameter to be transformed.
98 transform_params : dict
99 The metadata to be used for transformation. This passed to the
100 `transform` method of the sub-pipeline.
101
102 Returns
103 -------
104 transformed_value : object
105 The transformed value of the parameter.
106 """
107 if param_name not in cache:
108 # If the parameter is a tuple, transform each element of the
109 # tuple. This is needed to support the pattern present in
110 # `lightgbm` and `xgboost` where users can pass multiple
111 # validation sets.
112 if isinstance(param_value, tuple):
113 cache[param_name] = tuple(
114 sub_pipeline.transform(element, **transform_params)
115 for element in param_value
116 )
117 else:
118 cache[param_name] = sub_pipeline.transform(param_value, **transform_params)
119
120 return cache[param_name]

123class Pipeline(_BaseComposition):
124 """
125 A sequence of data transformers with an optional final predictor.
126
127 `Pipeline` allows you to sequentially apply a list of transformers to
128 preprocess the data and, if desired, conclude the sequence with a final
129 :term:`predictor` for predictive modeling.
130
131 Intermediate steps of the pipeline must be transformers, that is, they
132 must implement `fit` and `transform` methods.
133 The final :term:`estimator` only needs to implement `fit`.
134 The transformers in the pipeline can be cached using ``memory`` argument.
135
136 The purpose of the pipeline is to assemble several steps that can be
137 cross-validated together while setting different parameters. For this, it
138 enables setting parameters of the various steps using their names and the
139 parameter name separated by a `'__'`, as in the example below. A step's
140 estimator may be replaced entirely by setting the parameter with its name
141 to another estimator, or a transformer removed by setting it to
142 `'passthrough'` or `None`.
143
144 For an example use case of `Pipeline` combined with
145 :class:`~sklearn.model_selection.GridSearchCV`, refer to
146 :ref:`sphx_glr_auto_examples_compose_plot_compare_reduction.py`. The
147 example :ref:`sphx_glr_auto_examples_compose_plot_digits_pipe.py` shows how
148 to grid search on a pipeline using `'__'` as a separator in the parameter names.
149
150 Read more in the :ref:`User Guide <pipeline>`.
151
152 .. versionadded:: 0.5
153
154 Parameters
155 ----------
156 steps : list of tuples
157 List of (name of step, estimator) tuples that are to be chained in
158 sequential order. To be compatible with the scikit-learn API, all steps
159 must define `fit`. All non-last steps must also define `transform`. See
160 :ref:`Combining Estimators <combining_estimators>` for more details.
161
162 transform_input : list of str, default=None
163 The names of the :term:`metadata` parameters that should be transformed by the
164 pipeline before passing it to the step consuming it.
165
166 This enables transforming some input arguments to ``fit`` (other than ``X``)
167 to be transformed by the steps of the pipeline up to the step which requires
168 them. Requirement is defined via :ref:`metadata routing <metadata_routing>`.
169 For instance, this can be used to pass a validation set through the pipeline.
170
171 You can only set this if metadata routing is enabled, which you
172 can enable using ``sklearn.set_config(enable_metadata_routing=True)``.
173
174 .. versionadded:: 1.6
175
176 memory : str or object with the joblib.Memory interface, default=None
177 Used to cache the fitted transformers of the pipeline. The last step
178 will never be cached, even if it is a transformer. By default, no
179 caching is performed. If a string is given, it is the path to the
180 caching directory. Enabling caching triggers a clone of the transformers
181 before fitting. Therefore, the transformer instance given to the
182 pipeline cannot be inspected directly. Use the attribute ``named_steps``
183 or ``steps`` to inspect estimators within the pipeline. Caching the
184 transformers is advantageous when fitting is time consuming. See
185 :ref:`sphx_glr_auto_examples_neighbors_plot_caching_nearest_neighbors.py`
186 for an example on how to enable caching.
187
188 verbose : bool, default=False
189 If True, the time elapsed while fitting each step will be printed as it
190 is completed.
191
192 Attributes
193 ----------
194 named_steps : :class:`~sklearn.utils.Bunch`
195 Dictionary-like object, with the following attributes.
196 Read-only attribute to access any step parameter by user given name.
197 Keys are step names and values are steps parameters.
198
199 classes_ : ndarray of shape (n_classes,)
200 The classes labels. Only exist if the last step of the pipeline is a
201 classifier.
202
203 n_features_in_ : int
204 Number of features seen during :term:`fit`. Only defined if the
205 underlying first estimator in `steps` exposes such an attribute
206 when fit.
207
208 .. versionadded:: 0.24
209
210 feature_names_in_ : ndarray of shape (`n_features_in_`,)
211 Names of features seen during :term:`fit`. Only defined if the
212 underlying estimator exposes such an attribute when fit.
213
214 .. versionadded:: 1.0
215
216 See Also
217 --------
218 make_pipeline : Convenience function for simplified pipeline construction.
219
220 Examples
221 --------
222 >>> from sklearn.svm import SVC
223 >>> from sklearn.preprocessing import StandardScaler
224 >>> from sklearn.datasets import make_classification
225 >>> from sklearn.model_selection import train_test_split
226 >>> from sklearn.pipeline import Pipeline
227 >>> X, y = make_classification(random_state=0)
228 >>> X_train, X_test, y_train, y_test = train_test_split(X, y,
229 ... random_state=0)
230 >>> pipe = Pipeline([('scaler', StandardScaler()), ('svc', SVC())])
231 >>> # The pipeline can be used as any other estimator
232 >>> # and avoids leaking the test set into the train set
233 >>> pipe.fit(X_train, y_train).score(X_test, y_test)
234 0.88
235 >>> # An estimator's parameter can be set using '__' syntax
236 >>> pipe.set_params(svc__C=10).fit(X_train, y_train).score(X_test, y_test)
237 0.76
238 """

322 def _validate_steps(self):
323 if not self.steps:
324 raise ValueError("The pipeline is empty. Please add steps.")
325 names, estimators = zip(*self.steps)
326
327 # validate names
328 self._validate_names(names)
329
330 # validate estimators
331 transformers = estimators[:-1]
332 estimator = estimators[-1]
333
334 for t in transformers:
335 if t is None or t == "passthrough":
336 continue
337 if not (hasattr(t, "fit") or hasattr(t, "fit_transform")) or not hasattr(
338 t, "transform"
339 ):
340 raise TypeError(
341 "All intermediate steps should be "
342 "transformers and implement fit and transform "
343 "or be the string 'passthrough' "
344 "'%s' (type %s) doesn't" % (t, type(t))
345 )
346
347 # We allow last estimator to be None as an identity transformation
348 if (
349 estimator is not None
350 and estimator != "passthrough"
351 and not hasattr(estimator, "fit")
352 ):
353 raise TypeError(
354 "Last step of Pipeline should implement fit "
355 "or be the string 'passthrough'. "
356 "'%s' (type %s) doesn't" % (estimator, type(estimator))

359 def _iter(self, with_final=True, filter_passthrough=True):
360 """
361 Generate (idx, (name, trans)) tuples from self.steps
362
363 When filter_passthrough is True, 'passthrough' and None transformers
364 are filtered out.
365 """
366 stop = len(self.steps)
367 if not with_final:
368 stop -= 1
369
370 for idx, (name, trans) in enumerate(islice(self.steps, 0, stop)):
371 if not filter_passthrough:
372 yield idx, name, trans
373 elif trans is not None and trans != "passthrough":
374 yield idx, name, trans

446 def _check_method_params(self, method, props, **kwargs):
447 if _routing_enabled():
448 routed_params = process_routing(self, method, **props, **kwargs)
449 return routed_params
450 else:
451 fit_params_steps = Bunch(
452 **{
453 name: Bunch(**{method: {} for method in METHODS})
454 for name, step in self.steps
455 if step is not None
456 }
457 )
458 for pname, pval in props.items():
459 if "__" not in pname:
460 raise ValueError(
461 "Pipeline.fit does not accept the {} parameter. "
462 "You can pass parameters to specific steps of your "
463 "pipeline using the stepname__parameter format, e.g. "
464 "`Pipeline.fit(X, y, logisticregression__sample_weight"
465 "=sample_weight)`.".format(pname)
466 )
467 step, param = pname.split("__", 1)
468 fit_params_steps[step]["fit"][param] = pval
469 # without metadata routing, fit_transform and fit_predict
470 # get all the same params and pass it to the last fit.
471 fit_params_steps[step]["fit_transform"][param] = pval
472 fit_params_steps[step]["fit_predict"][param] = pval
473 return fit_params_steps

475 def _get_metadata_for_step(self, *, step_idx, step_params, all_params):
476 """Get params (metadata) for step `name`.
477
478 This transforms the metadata up to this step if required, which is
479 indicated by the `transform_input` parameter.
480
481 If a param in `step_params` is included in the `transform_input` list,
482 it will be transformed.
483
484 Parameters
485 ----------
486 step_idx : int
487 Index of the step in the pipeline.
488
489 step_params : dict
490 Parameters specific to the step. These are routed parameters, e.g.
491 `routed_params[name]`. If a parameter name here is included in the
492 `pipeline.transform_input`, then it will be transformed. Note that
493 these parameters are *after* routing, so the aliases are already
494 resolved.
495
496 all_params : dict
497 All parameters passed by the user. Here this is used to call
498 `transform` on the slice of the pipeline itself.
499
500 Returns
501 -------
502 dict
503 Parameters to be passed to the step. The ones which should be
504 transformed are transformed.
505 """
506 if (
507 self.transform_input is None
508 or not all_params
509 or not step_params
510 or step_idx == 0
511 ):
512 # we only need to process step_params if transform_input is set
513 # and metadata is given by the user.
514 return step_params
515
516 sub_pipeline = self[:step_idx]
517 sub_metadata_routing = get_routing_for_object(sub_pipeline)
518 # here we get the metadata required by sub_pipeline.transform
519 transform_params = {
520 key: value
521 for key, value in all_params.items()
522 if key
523 in sub_metadata_routing.consumes(
524 method="transform", params=all_params.keys()
525 )
526 }
527 transformed_params = dict() # this is to be returned
528 transformed_cache = dict() # used to transform each param once
529 # `step_params` is the output of `process_routing`, so it has a dict for each
530 # method (e.g. fit, transform, predict), which are the args to be passed to
531 # those methods. We need to transform the parameters which are in the
532 # `transform_input`, before returning these dicts.
533 for method, method_params in step_params.items():
534 transformed_params[method] = Bunch()
535 for param_name, param_value in method_params.items():
536 # An example of `(param_name, param_value)` is
537 # `('sample_weight', array([0.5, 0.5, ...]))`
538 if param_name in self.transform_input:
539 # This parameter now needs to be transformed by the sub_pipeline, to
540 # this step. We cache these computations to avoid repeating them.
541 transformed_params[method][param_name] = _cached_transform(
542 sub_pipeline,
543 cache=transformed_cache,
544 param_name=param_name,
545 param_value=param_value,
546 transform_params=transform_params,
547 )
548 else:
549 transformed_params[method][param_name] = param_value
550 return transformed_params

554 def _fit(self, X, y=None, routed_params=None, raw_params=None):
555 """Fit the pipeline except the last step.
556
557 routed_params is the output of `process_routing`
558 raw_params is the parameters passed by the user, used when `transform_input`
559 is set by the user, to transform metadata using a sub-pipeline.
560 """
561 # shallow copy of steps - this should really be steps_
562 self.steps = list(self.steps)
563 self._validate_steps()
564 # Setup the memory
565 memory = check_memory(self.memory)
566
567 fit_transform_one_cached = memory.cache(_fit_transform_one)
568
569 for step_idx, name, transformer in self._iter(
570 with_final=False, filter_passthrough=False
571 ):
572 if transformer is None or transformer == "passthrough":
573 with _print_elapsed_time("Pipeline", self._log_message(step_idx)):
574 continue
575
576 if hasattr(memory, "location") and memory.location is None:
577 # we do not clone when caching is disabled to
578 # preserve backward compatibility
579 cloned_transformer = transformer
580 else:
581 cloned_transformer = clone(transformer)
582 # Fit or load from cache the current transformer
583 step_params = self._get_metadata_for_step(
584 step_idx=step_idx,
585 step_params=routed_params[name],
586 all_params=raw_params,
587 )
588
589 X, fitted_transformer = fit_transform_one_cached(
590 cloned_transformer,
591 X,
592 y,
593 weight=None,
594 message_clsname="Pipeline",
595 message=self._log_message(step_idx),
596 params=step_params,
597 )
598 # Replace the transformer of the step with the fitted
599 # transformer. This is necessary when loading the transformer
600 # from the cache.
601 self.steps[step_idx] = (name, fitted_transformer)
602 return X

608 def fit(self, X, y=None, **params):
609 """Fit the model.
610
611 Fit all the transformers one after the other and sequentially transform the
612 data. Finally, fit the transformed data using the final estimator.
613
614 Parameters
615 ----------
616 X : iterable
617 Training data. Must fulfill input requirements of first step of the
618 pipeline.
619
620 y : iterable, default=None
621 Training targets. Must fulfill label requirements for all steps of
622 the pipeline.
623
624 **params : dict of str -> object
625 - If `enable_metadata_routing=False` (default): Parameters passed to the
626 ``fit`` method of each step, where each parameter name is prefixed such
627 that parameter ``p`` for step ``s`` has key ``s__p``.
628
629 - If `enable_metadata_routing=True`: Parameters requested and accepted by
630 steps. Each step must have requested certain metadata for these parameters
631 to be forwarded to them.
632
633 .. versionchanged:: 1.4
634 Parameters are now passed to the ``transform`` method of the
635 intermediate steps as well, if requested, and if
636 `enable_metadata_routing=True` is set via
637 :func:`~sklearn.set_config`.
638
639 See :ref:`Metadata Routing User Guide <metadata_routing>` for more
640 details.
641
642 Returns
643 -------
644 self : object
645 Pipeline with fitted steps.
646 """
647 if not _routing_enabled() and self.transform_input is not None:
648 raise ValueError(
649 "The `transform_input` parameter can only be set if metadata "
650 "routing is enabled. You can enable metadata routing using "
651 "`sklearn.set_config(enable_metadata_routing=True)`."
652 )
653
654 routed_params = self._check_method_params(method="fit", props=params)
655 Xt = self._fit(X, y, routed_params, raw_params=params)
656 with _print_elapsed_time("Pipeline", self._log_message(len(self.steps) - 1)):
657 if self._final_estimator != "passthrough":
658 last_step_params = self._get_metadata_for_step(
659 step_idx=len(self) - 1,
660 step_params=routed_params[self.steps[-1][0]],
661 all_params=params,
662 )
663 self._final_estimator.fit(Xt, y, **last_step_params["fit"])
664
665 return self

679 def fit_transform(self, X, y=None, **params):
680 """Fit the model and transform with the final estimator.
681
682 Fit all the transformers one after the other and sequentially transform
683 the data. Only valid if the final estimator either implements
684 `fit_transform` or `fit` and `transform`.
685
686 Parameters
687 ----------
688 X : iterable
689 Training data. Must fulfill input requirements of first step of the
690 pipeline.
691
692 y : iterable, default=None
693 Training targets. Must fulfill label requirements for all steps of
694 the pipeline.
695
696 **params : dict of str -> object
697 - If `enable_metadata_routing=False` (default): Parameters passed to the
698 ``fit`` method of each step, where each parameter name is prefixed such
699 that parameter ``p`` for step ``s`` has key ``s__p``.
700
701 - If `enable_metadata_routing=True`: Parameters requested and accepted by
702 steps. Each step must have requested certain metadata for these parameters
703 to be forwarded to them.
704
705 .. versionchanged:: 1.4
706 Parameters are now passed to the ``transform`` method of the
707 intermediate steps as well, if requested, and if
708 `enable_metadata_routing=True`.
709
710 See :ref:`Metadata Routing User Guide <metadata_routing>` for more
711 details.
712
713 Returns
714 -------
715 Xt : ndarray of shape (n_samples, n_transformed_features)
716 Transformed samples.
717 """
718 routed_params = self._check_method_params(method="fit_transform", props=params)
719 Xt = self._fit(X, y, routed_params)
720
721 last_step = self._final_estimator
722 with _print_elapsed_time("Pipeline", self._log_message(len(self.steps) - 1)):
723 if last_step == "passthrough":
724 return Xt
725 last_step_params = self._get_metadata_for_step(
726 step_idx=len(self) - 1,
727 step_params=routed_params[self.steps[-1][0]],
728 all_params=params,
729 )
730 if hasattr(last_step, "fit_transform"):
731 return last_step.fit_transform(
732 Xt, y, **last_step_params["fit_transform"]
733 )
734 else:
735 return last_step.fit(Xt, y, **last_step_params["fit"]).transform(
736 Xt, **last_step_params["transform"]

1286 def __sklearn_is_fitted__(self):
1287 """Indicate whether pipeline has been fit.
1288
1289 This is done by checking whether the last non-`passthrough` step of the
1290 pipeline is fitted.
1291
1292 An empty pipeline is considered fitted.
1293 """
1294 # First find the last step that is not 'passthrough'
1295 last_step = None
1296 for _, estimator in reversed(self.steps):
1297 if estimator != "passthrough":
1298 last_step = estimator
1299 break
1300
1301 if last_step is None:
1302 # All steps are 'passthrough', so the pipeline is considered fitted
1303 return True
1304
1305 try:
1306 # check if the last step of the pipeline is fitted
1307 # we only check the last step since if the last step is fit, it
1308 # means the previous steps should also be fit. This is faster than
1309 # checking if every step of the pipeline is fit.
1310 check_is_fitted(last_step)
1311 return True
1312 except NotFittedError:
1313 return False

1334 def get_metadata_routing(self):
1335 """Get metadata routing of this object.
1336
1337 Please check :ref:`User Guide <metadata_routing>` on how the routing
1338 mechanism works.
1339
1340 Returns
1341 -------
1342 routing : MetadataRouter
1343 A :class:`~sklearn.utils.metadata_routing.MetadataRouter` encapsulating
1344 routing information.
1345 """
1346 router = MetadataRouter(owner=self.__class__.__name__)
1347
1348 # first we add all steps except the last one
1349 for _, name, trans in self._iter(with_final=False, filter_passthrough=True):
1350 method_mapping = MethodMapping()
1351 # fit, fit_predict, and fit_transform call fit_transform if it
1352 # exists, or else fit and transform
1353 if hasattr(trans, "fit_transform"):
1354 (
1355 method_mapping.add(caller="fit", callee="fit_transform")
1356 .add(caller="fit_transform", callee="fit_transform")
1357 .add(caller="fit_predict", callee="fit_transform")
1358 )
1359 else:
1360 (
1361 method_mapping.add(caller="fit", callee="fit")
1362 .add(caller="fit", callee="transform")
1363 .add(caller="fit_transform", callee="fit")
1364 .add(caller="fit_transform", callee="transform")
1365 .add(caller="fit_predict", callee="fit")
1366 .add(caller="fit_predict", callee="transform")
1367 )
1368
1369 (
1370 method_mapping.add(caller="predict", callee="transform")
1371 .add(caller="predict", callee="transform")
1372 .add(caller="predict_proba", callee="transform")
1373 .add(caller="decision_function", callee="transform")
1374 .add(caller="predict_log_proba", callee="transform")
1375 .add(caller="transform", callee="transform")
1376 .add(caller="inverse_transform", callee="inverse_transform")
1377 .add(caller="score", callee="transform")
1378 )
1379
1380 router.add(method_mapping=method_mapping, **{name: trans})
1381
1382 final_name, final_est = self.steps[-1]
1383 if final_est is None or final_est == "passthrough":
1384 return router
1385
1386 # then we add the last step
1387 method_mapping = MethodMapping()
1388 if hasattr(final_est, "fit_transform"):
1389 method_mapping.add(caller="fit_transform", callee="fit_transform")
1390 else:
1391 method_mapping.add(caller="fit", callee="fit").add(
1392 caller="fit", callee="transform"
1393 )
1394 (
1395 method_mapping.add(caller="fit", callee="fit")
1396 .add(caller="predict", callee="predict")
1397 .add(caller="fit_predict", callee="fit_predict")
1398 .add(caller="predict_proba", callee="predict_proba")
1399 .add(caller="decision_function", callee="decision_function")
1400 .add(caller="predict_log_proba", callee="predict_log_proba")
1401 .add(caller="transform", callee="transform")
1402 .add(caller="inverse_transform", callee="inverse_transform")
1403 .add(caller="score", callee="score")
1404 )
1405

1410def _name_estimators(estimators):
1411 """Generate names for estimators."""
1412
1413 names = [
1414 estimator if isinstance(estimator, str) else type(estimator).__name__.lower()
1415 for estimator in estimators
1416 ]
1417 namecount = defaultdict(int)
1418 for est, name in zip(estimators, names):
1419 namecount[name] += 1
1420
1421 for k, v in list(namecount.items()):
1422 if v == 1:
1423 del namecount[k]
1424
1425 for i in reversed(range(len(estimators))):
1426 name = names[i]
1427 if name in namecount:
1428 names[i] += "-%d" % namecount[name]
1429 namecount[name] -= 1
1430
1431 return list(zip(names, estimators))

1434def make_pipeline(*steps, memory=None, transform_input=None, verbose=False):
1435 """Construct a :class:`Pipeline` from the given estimators.
1436
1437 This is a shorthand for the :class:`Pipeline` constructor; it does not
1438 require, and does not permit, naming the estimators. Instead, their names
1439 will be set to the lowercase of their types automatically.
1440
1441 Parameters
1442 ----------
1443 *steps : list of Estimator objects
1444 List of the scikit-learn estimators that are chained together.
1445
1446 memory : str or object with the joblib.Memory interface, default=None
1447 Used to cache the fitted transformers of the pipeline. The last step
1448 will never be cached, even if it is a transformer. By default, no
1449 caching is performed. If a string is given, it is the path to the
1450 caching directory. Enabling caching triggers a clone of the transformers
1451 before fitting. Therefore, the transformer instance given to the
1452 pipeline cannot be inspected directly. Use the attribute ``named_steps``
1453 or ``steps`` to inspect estimators within the pipeline. Caching the
1454 transformers is advantageous when fitting is time consuming.

1457 This enables transforming some input arguments to ``fit`` (other than ``X``)
1458 to be transformed by the steps of the pipeline up to the step which requires
1459 them. Requirement is defined via :ref:`metadata routing <metadata_routing>`.
1460 This can be used to pass a validation set through the pipeline for instance.
1461
1462 You can only set this if metadata routing is enabled, which you
1463 can enable using ``sklearn.set_config(enable_metadata_routing=True)``.
1464
1465 .. versionadded:: 1.6
1466
1467 verbose : bool, default=False
1468 If True, the time elapsed while fitting each step will be printed as it
1469 is completed.
1470
1471 Returns
1472 -------
1473 p : Pipeline
1474 Returns a scikit-learn :class:`Pipeline` object.
1475
1476 See Also
1477 --------
1478 Pipeline : Class for creating a pipeline of transforms with a final
1479 estimator.
1480
1481 Examples
1482 --------
1483 >>> from sklearn.naive_bayes import GaussianNB
1484 >>> from sklearn.preprocessing import StandardScaler
1485 >>> from sklearn.pipeline import make_pipeline

1488 ('gaussiannb', GaussianNB())])
