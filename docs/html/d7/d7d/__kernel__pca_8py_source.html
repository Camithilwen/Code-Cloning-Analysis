32class KernelPCA(ClassNamePrefixFeaturesOutMixin, TransformerMixin, BaseEstimator):
33 """Kernel Principal component analysis (KPCA).
34
35 Non-linear dimensionality reduction through the use of kernels [1]_, see also
36 :ref:`metrics`.
37
38 It uses the :func:`scipy.linalg.eigh` LAPACK implementation of the full SVD
39 or the :func:`scipy.sparse.linalg.eigsh` ARPACK implementation of the
40 truncated SVD, depending on the shape of the input data and the number of
41 components to extract. It can also use a randomized truncated SVD by the
42 method proposed in [3]_, see `eigen_solver`.
43
44 For a usage example and comparison between
45 Principal Components Analysis (PCA) and its kernelized version (KPCA), see
46 :ref:`sphx_glr_auto_examples_decomposition_plot_kernel_pca.py`.
47
48 For a usage example in denoising images using KPCA, see
49 :ref:`sphx_glr_auto_examples_applications_plot_digits_denoising.py`.
50
51 Read more in the :ref:`User Guide <kernel_PCA>`.
52
53 Parameters
54 ----------
55 n_components : int, default=None
56 Number of components. If None, all non-zero components are kept.
57
58 kernel : {'linear', 'poly', 'rbf', 'sigmoid', 'cosine', 'precomputed'} \
59 or callable, default='linear'
60 Kernel used for PCA.
61
62 gamma : float, default=None
63 Kernel coefficient for rbf, poly and sigmoid kernels. Ignored by other
64 kernels. If ``gamma`` is ``None``, then it is set to ``1/n_features``.
65
66 degree : float, default=3
67 Degree for poly kernels. Ignored by other kernels.
68
69 coef0 : float, default=1
70 Independent term in poly and sigmoid kernels.
71 Ignored by other kernels.
72
73 kernel_params : dict, default=None
74 Parameters (keyword arguments) and
75 values for kernel passed as callable object.
76 Ignored by other kernels.
77
78 alpha : float, default=1.0
79 Hyperparameter of the ridge regression that learns the
80 inverse transform (when fit_inverse_transform=True).
81
82 fit_inverse_transform : bool, default=False
83 Learn the inverse transform for non-precomputed kernels
84 (i.e. learn to find the pre-image of a point). This method is based
85 on [2]_.
86
87 eigen_solver : {'auto', 'dense', 'arpack', 'randomized'}, \
88 default='auto'
89 Select eigensolver to use. If `n_components` is much
90 less than the number of training samples, randomized (or arpack to a
91 smaller extent) may be more efficient than the dense eigensolver.
92 Randomized SVD is performed according to the method of Halko et al
93 [3]_.
94
95 auto :
96 the solver is selected by a default policy based on n_samples
97 (the number of training samples) and `n_components`:
98 if the number of components to extract is less than 10 (strict) and
99 the number of samples is more than 200 (strict), the 'arpack'
100 method is enabled. Otherwise the exact full eigenvalue
101 decomposition is computed and optionally truncated afterwards
102 ('dense' method).
103 dense :
104 run exact full eigenvalue decomposition calling the standard
105 LAPACK solver via `scipy.linalg.eigh`, and select the components
106 by postprocessing
107 arpack :
108 run SVD truncated to n_components calling ARPACK solver using
109 `scipy.sparse.linalg.eigsh`. It requires strictly
110 0 < n_components < n_samples
111 randomized :
112 run randomized SVD by the method of Halko et al. [3]_. The current
113 implementation selects eigenvalues based on their module; therefore
114 using this method can lead to unexpected results if the kernel is
115 not positive semi-definite. See also [4]_.
116
117 .. versionchanged:: 1.0
118 `'randomized'` was added.
119
120 tol : float, default=0
121 Convergence tolerance for arpack.
122 If 0, optimal value will be chosen by arpack.
123
124 max_iter : int, default=None
125 Maximum number of iterations for arpack.
126 If None, optimal value will be chosen by arpack.
127
128 iterated_power : int >= 0, or 'auto', default='auto'
129 Number of iterations for the power method computed by
130 svd_solver == 'randomized'. When 'auto', it is set to 7 when
131 `n_components < 0.1 * min(X.shape)`, other it is set to 4.
132
133 .. versionadded:: 1.0
134
135 remove_zero_eig : bool, default=False
136 If True, then all components with zero eigenvalues are removed, so
137 that the number of components in the output may be < n_components
138 (and sometimes even zero due to numerical instability).
139 When n_components is None, this parameter is ignored and components
140 with zero eigenvalues are removed regardless.
141
142 random_state : int, RandomState instance or None, default=None
143 Used when ``eigen_solver`` == 'arpack' or 'randomized'. Pass an int
144 for reproducible results across multiple function calls.
145 See :term:`Glossary <random_state>`.
146
147 .. versionadded:: 0.18
148
149 copy_X : bool, default=True
150 If True, input X is copied and stored by the model in the `X_fit_`
151 attribute. If no further changes will be done to X, setting
152 `copy_X=False` saves memory by storing a reference.
153
154 .. versionadded:: 0.18
155
156 n_jobs : int, default=None
157 The number of parallel jobs to run.
158 ``None`` means 1 unless in a :obj:`joblib.parallel_backend` context.
159 ``-1`` means using all processors. See :term:`Glossary <n_jobs>`
160 for more details.
161
162 .. versionadded:: 0.18
163
164 Attributes
165 ----------
166 eigenvalues_ : ndarray of shape (n_components,)
167 Eigenvalues of the centered kernel matrix in decreasing order.
168 If `n_components` and `remove_zero_eig` are not set,
169 then all values are stored.
170
171 eigenvectors_ : ndarray of shape (n_samples, n_components)
172 Eigenvectors of the centered kernel matrix. If `n_components` and
173 `remove_zero_eig` are not set, then all components are stored.
174
175 dual_coef_ : ndarray of shape (n_samples, n_features)
176 Inverse transform matrix. Only available when
177 ``fit_inverse_transform`` is True.
178
179 X_transformed_fit_ : ndarray of shape (n_samples, n_components)
180 Projection of the fitted data on the kernel principal components.
181 Only available when ``fit_inverse_transform`` is True.
182
183 X_fit_ : ndarray of shape (n_samples, n_features)
184 The data used to fit the model. If `copy_X=False`, then `X_fit_` is
185 a reference. This attribute is used for the calls to transform.
186
187 n_features_in_ : int
188 Number of features seen during :term:`fit`.
189
190 .. versionadded:: 0.24
191
192 feature_names_in_ : ndarray of shape (`n_features_in_`,)
193 Names of features seen during :term:`fit`. Defined only when `X`
194 has feature names that are all strings.
195
196 .. versionadded:: 1.0
197
198 gamma_ : float
199 Kernel coefficient for rbf, poly and sigmoid kernels. When `gamma`
200 is explicitly provided, this is just the same as `gamma`. When `gamma`
201 is `None`, this is the actual value of kernel coefficient.
202

319 def _get_kernel(self, X, Y=None):
320 if callable(self.kernel):
321 params = self.kernel_params or {}
322 else:
323 params = {"gamma": self.gamma_, "degree": self.degree, "coef0": self.coef0}
324 return pairwise_kernels(
325 X, Y, metric=self.kernel, filter_params=True, n_jobs=self.n_jobs, **params

328 def _fit_transform_in_place(self, K):
329 """Fit's using kernel K"""
330 # center kernel in place
331 K = self._centerer.fit(K).transform(K, copy=False)
332
333 # adjust n_components according to user inputs
334 if self.n_components is None:
335 n_components = K.shape[0] # use all dimensions
336 else:
337 n_components = min(K.shape[0], self.n_components)
338
339 # compute eigenvectors
340 if self.eigen_solver == "auto":
341 if K.shape[0] > 200 and n_components < 10:
342 eigen_solver = "arpack"
343 else:
344 eigen_solver = "dense"
345 else:
346 eigen_solver = self.eigen_solver
347
348 if eigen_solver == "dense":
349 # Note: subset_by_index specifies the indices of smallest/largest to return
350 self.eigenvalues_, self.eigenvectors_ = eigh(
351 K, subset_by_index=(K.shape[0] - n_components, K.shape[0] - 1)
352 )
353 elif eigen_solver == "arpack":
354 v0 = _init_arpack_v0(K.shape[0], self.random_state)
355 self.eigenvalues_, self.eigenvectors_ = eigsh(
356 K, n_components, which="LA", tol=self.tol, maxiter=self.max_iter, v0=v0
357 )
358 elif eigen_solver == "randomized":
359 self.eigenvalues_, self.eigenvectors_ = _randomized_eigsh(
360 K,
361 n_components=n_components,
362 n_iter=self.iterated_power,
363 random_state=self.random_state,
364 selection="module",
365 )
366
367 # make sure that the eigenvalues are ok and fix numerical issues
368 self.eigenvalues_ = _check_psd_eigenvalues(
369 self.eigenvalues_, enable_warnings=False
370 )
371
372 # flip eigenvectors' sign to enforce deterministic output
373 self.eigenvectors_, _ = svd_flip(u=self.eigenvectors_, v=None)
374
375 # sort eigenvectors in descending order
376 indices = self.eigenvalues_.argsort()[::-1]
377 self.eigenvalues_ = self.eigenvalues_[indices]
378 self.eigenvectors_ = self.eigenvectors_[:, indices]
379
380 # remove eigenvectors with a zero eigenvalue (null space) if required
381 if self.remove_zero_eig or self.n_components is None:
382 self.eigenvectors_ = self.eigenvectors_[:, self.eigenvalues_ > 0]
383 self.eigenvalues_ = self.eigenvalues_[self.eigenvalues_ > 0]
384
385 # Maintenance note on Eigenvectors normalization
386 # ----------------------------------------------
387 # there is a link between
388 # the eigenvectors of K=Phi(X)'Phi(X) and the ones of Phi(X)Phi(X)'
389 # if v is an eigenvector of K
390 # then Phi(X)v is an eigenvector of Phi(X)Phi(X)'
391 # if u is an eigenvector of Phi(X)Phi(X)'
392 # then Phi(X)'u is an eigenvector of Phi(X)'Phi(X)
393 #
394 # At this stage our self.eigenvectors_ (the v) have norm 1, we need to scale
395 # them so that eigenvectors in kernel feature space (the u) have norm=1
396 # instead
397 #
398 # We COULD scale them here:
399 # self.eigenvectors_ = self.eigenvectors_ / np.sqrt(self.eigenvalues_)
400 #
401 # But choose to perform that LATER when needed, in `fit()` and in
402 # `transform()`.
403
404 return K

406 def _fit_inverse_transform(self, X_transformed, X):
407 if hasattr(X, "tocsr"):
408 raise NotImplementedError(
409 "Inverse transform not implemented for sparse matrices!"
410 )
411
412 n_samples = X_transformed.shape[0]
413 K = self._get_kernel(X_transformed)
414 K.flat[:: n_samples + 1] += self.alpha
415 self.dual_coef_ = linalg.solve(K, X, assume_a="pos", overwrite_a=True)
416 self.X_transformed_fit_ = X_transformed

418 @_fit_context(prefer_skip_nested_validation=True)
419 def fit(self, X, y=None):
420 """Fit the model from data in X.
421
422 Parameters
423 ----------
424 X : {array-like, sparse matrix} of shape (n_samples, n_features)
425 Training vector, where `n_samples` is the number of samples
426 and `n_features` is the number of features.
427
428 y : Ignored
429 Not used, present for API consistency by convention.
430
431 Returns
432 -------
433 self : object
434 Returns the instance itself.
435 """
436 if self.fit_inverse_transform and self.kernel == "precomputed":
437 raise ValueError("Cannot fit_inverse_transform with a precomputed kernel.")
438 X = validate_data(self, X, accept_sparse="csr", copy=self.copy_X)
439 self.gamma_ = 1 / X.shape[1] if self.gamma is None else self.gamma
440 self._centerer = KernelCenterer().set_output(transform="default")
441 K = self._get_kernel(X)
442 # When kernel="precomputed", K is X but it's safe to perform in place operations
443 # on K because a copy was made before if requested by copy_X.
444 self._fit_transform_in_place(K)
445
446 if self.fit_inverse_transform:
447 # no need to use the kernel to transform X, use shortcut expression
448 X_transformed = self.eigenvectors_ * np.sqrt(self.eigenvalues_)
449
450 self._fit_inverse_transform(X_transformed, X)
451
452 self.X_fit_ = X
453 return self

455 def fit_transform(self, X, y=None, **params):
456 """Fit the model from data in X and transform X.
457
458 Parameters
459 ----------
460 X : {array-like, sparse matrix} of shape (n_samples, n_features)
461 Training vector, where `n_samples` is the number of samples
462 and `n_features` is the number of features.
463
464 y : Ignored
465 Not used, present for API consistency by convention.
466
467 **params : kwargs
468 Parameters (keyword arguments) and values passed to
469 the fit_transform instance.
470
471 Returns
472 -------
473 X_new : ndarray of shape (n_samples, n_components)
474 Returns the instance itself.
475 """
476 self.fit(X, **params)
477
478 # no need to use the kernel to transform X, use shortcut expression
479 X_transformed = self.eigenvectors_ * np.sqrt(self.eigenvalues_)
480
481 if self.fit_inverse_transform:
482 self._fit_inverse_transform(X_transformed, X)
483
484 return X_transformed

486 def transform(self, X):
487 """Transform X.
488
489 Parameters
490 ----------
491 X : {array-like, sparse matrix} of shape (n_samples, n_features)
492 Training vector, where `n_samples` is the number of samples
493 and `n_features` is the number of features.
494
495 Returns
496 -------
497 X_new : ndarray of shape (n_samples, n_components)
498 Returns the instance itself.
499 """
500 check_is_fitted(self)
501 X = validate_data(self, X, accept_sparse="csr", reset=False)
502
503 # Compute centered gram matrix between X and training data X_fit_
504 K = self._centerer.transform(self._get_kernel(X, self.X_fit_))
505
506 # scale eigenvectors (properly account for null-space for dot product)
507 non_zeros = np.flatnonzero(self.eigenvalues_)
508 scaled_alphas = np.zeros_like(self.eigenvectors_)
509 scaled_alphas[:, non_zeros] = self.eigenvectors_[:, non_zeros] / np.sqrt(
510 self.eigenvalues_[non_zeros]
511 )
512
513 # Project with a scalar product between K and the scaled eigenvectors
514 return np.dot(K, scaled_alphas)

516 def inverse_transform(self, X):
517 """Transform X back to original space.
518
519 ``inverse_transform`` approximates the inverse transformation using
520 a learned pre-image. The pre-image is learned by kernel ridge
521 regression of the original data on their low-dimensional representation
522 vectors.
523
524 .. note:
525 :meth:`~sklearn.decomposition.fit` internally uses a centered
526 kernel. As the centered kernel no longer contains the information
527 of the mean of kernel features, such information is not taken into
528 account in reconstruction.
529
530 .. note::
531 When users want to compute inverse transformation for 'linear'
532 kernel, it is recommended that they use
533 :class:`~sklearn.decomposition.PCA` instead. Unlike
534 :class:`~sklearn.decomposition.PCA`,
535 :class:`~sklearn.decomposition.KernelPCA`'s ``inverse_transform``
536 does not reconstruct the mean of data when 'linear' kernel is used
537 due to the use of centered kernel.
538
539 Parameters
540 ----------
541 X : {array-like, sparse matrix} of shape (n_samples, n_components)
542 Training vector, where `n_samples` is the number of samples
543 and `n_features` is the number of features.
544
545 Returns
546 -------
547 X_original : ndarray of shape (n_samples, n_features)
548 Returns the instance itself.
549
550 References
551 ----------
552 `Bakır, Gökhan H., Jason Weston, and Bernhard Schölkopf.
553 "Learning to find pre-images."
554 Advances in neural information processing systems 16 (2004): 449-456.
555 <https://papers.nips.cc/paper/2003/file/ac1ad983e08ad3304a97e147f522747e-Paper.pdf>`_
556 """
557 if not self.fit_inverse_transform:
558 raise NotFittedError(
559 "The fit_inverse_transform parameter was not"
560 " set to True when instantiating and hence "
561 "the inverse transform is not available."
562 )
563
564 K = self._get_kernel(X, self.X_transformed_fit_)
565 return np.dot(K, self.dual_coef_)
