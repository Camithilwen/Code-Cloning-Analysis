1"""
2Support for plotting vector fields.
3
4Presently this contains Quiver and Barb. Quiver plots an arrow in the
5direction of the vector, with the size of the arrow related to the

30_quiver_doc = """
31Plot a 2D field of arrows.
32
33Call signature::
34
35 quiver([X, Y], U, V, [C], /, **kwargs)
36
37*X*, *Y* define the arrow locations, *U*, *V* define the arrow directions, and
38*C* optionally sets the color. The arguments *X*, *Y*, *U*, *V*, *C* are
39positional-only.
40
41**Arrow length**
42
43The default settings auto-scales the length of the arrows to a reasonable size.
44To change this behavior see the *scale* and *scale_units* parameters.
45
46**Arrow shape**
47
48The arrow shape is determined by *width*, *headwidth*, *headlength* and
49*headaxislength*. See the notes below.
50
51**Arrow styling**
52
53Each arrow is internally represented by a filled polygon with a default edge
54linewidth of 0. As a result, an arrow is rather a filled area, not a line with
55a head, and `.PolyCollection` properties like *linewidth*, *edgecolor*,
56*facecolor*, etc. act accordingly.
57
58
59Parameters
60----------
61X, Y : 1D or 2D array-like, optional
62 The x and y coordinates of the arrow locations.
63
64 If not given, they will be generated as a uniform integer meshgrid based
65 on the dimensions of *U* and *V*.
66
67 If *X* and *Y* are 1D but *U*, *V* are 2D, *X*, *Y* are expanded to 2D
68 using ``X, Y = np.meshgrid(X, Y)``. In this case ``len(X)`` and ``len(Y)``
69 must match the column and row dimensions of *U* and *V*.
70
71U, V : 1D or 2D array-like
72 The x and y direction components of the arrow vectors. The interpretation
73 of these components (in data or in screen space) depends on *angles*.
74
75 *U* and *V* must have the same number of elements, matching the number of
76 arrow locations in *X*, *Y*. *U* and *V* may be masked. Locations masked
77 in any of *U*, *V*, and *C* will not be drawn.
78
79C : 1D or 2D array-like, optional
80 Numeric data that defines the arrow colors by colormapping via *norm* and
81 *cmap*.
82
83 This does not support explicit colors. If you want to set colors directly,
84 use *color* instead. The size of *C* must match the number of arrow
85 locations.
86
87angles : {'uv', 'xy'} or array-like, default: 'uv'
88 Method for determining the angle of the arrows.
89
90 - 'uv': Arrow directions are based on
91 :ref:`display coordinates <coordinate-systems>`; i.e. a 45° angle will
92 always show up as diagonal on the screen, irrespective of figure or Axes
93 aspect ratio or Axes data ranges. This is useful when the arrows represent
94 a quantity whose direction is not tied to the x and y data coordinates.
95
96 If *U* == *V* the orientation of the arrow on the plot is 45 degrees
97 counter-clockwise from the horizontal axis (positive to the right).
98
99 - 'xy': Arrow direction in data coordinates, i.e. the arrows point from
100 (x, y) to (x+u, y+v). This is ideal for vector fields or gradient plots
101 where the arrows should directly represent movements or gradients in the
102 x and y directions.
103
104 - Arbitrary angles may be specified explicitly as an array of values
105 in degrees, counter-clockwise from the horizontal axis.
106
107 In this case *U*, *V* is only used to determine the length of the
108 arrows.
109
110 For example, ``angles=[30, 60, 90]`` will orient the arrows at 30, 60, and 90
111 degrees respectively, regardless of the *U* and *V* components.
112
113 Note: inverting a data axis will correspondingly invert the
114 arrows only with ``angles='xy'``.
115
116pivot : {'tail', 'mid', 'middle', 'tip'}, default: 'tail'
117 The part of the arrow that is anchored to the *X*, *Y* grid. The arrow
118 rotates about this point.
119
120 'mid' is a synonym for 'middle'.
121
122scale : float, optional
123 Scales the length of the arrow inversely.
124
125 Number of data values represented by one unit of arrow length on the plot.
126 For example, if the data represents velocity in meters per second (m/s), the
127 scale parameter determines how many meters per second correspond to one unit of
128 arrow length relative to the width of the plot.
129 Smaller scale parameter makes the arrow longer.
130
131 By default, an autoscaling algorithm is used to scale the arrow length to a
132 reasonable size, which is based on the average vector length and the number of
133 vectors.
134
135 The arrow length unit is given by the *scale_units* parameter.
136
137scale_units : {'width', 'height', 'dots', 'inches', 'x', 'y', 'xy'}, default: 'width'
138
139 The physical image unit, which is used for rendering the scaled arrow data *U*, *V*.
140
141 The rendered arrow length is given by
142
143 length in x direction = $\\frac{u}{\\mathrm{scale}} \\mathrm{scale_unit}$
144
145 length in y direction = $\\frac{v}{\\mathrm{scale}} \\mathrm{scale_unit}$
146
147 For example, ``(u, v) = (0.5, 0)`` with ``scale=10, scale_unit="width"`` results
148 in a horizontal arrow with a length of *0.5 / 10 * "width"*, i.e. 0.05 times the
149 Axes width.
150
151 Supported values are:
152
153 - 'width' or 'height': The arrow length is scaled relative to the width or height
154 of the Axes.
155 For example, ``scale_units='width', scale=1.0``, will result in an arrow length
156 of width of the Axes.
157
158 - 'dots': The arrow length of the arrows is in measured in display dots (pixels).
159
160 - 'inches': Arrow lengths are scaled based on the DPI (dots per inch) of the figure.
161 This ensures that the arrows have a consistent physical size on the figure,
162 in inches, regardless of data values or plot scaling.
163 For example, ``(u, v) = (1, 0)`` with ``scale_units='inches', scale=2`` results
164 in a 0.5 inch-long arrow.
165
166 - 'x' or 'y': The arrow length is scaled relative to the x or y axis units.
167 For example, ``(u, v) = (0, 1)`` with ``scale_units='x', scale=1`` results
168 in a vertical arrow with the length of 1 x-axis unit.
169
170 - 'xy': Arrow length will be same as 'x' or 'y' units.
171 This is useful for creating vectors in the x-y plane where u and v have
172 the same units as x and y. To plot vectors in the x-y plane with u and v having
173 the same units as x and y, use ``angles='xy', scale_units='xy', scale=1``.
174
175 Note: Setting *scale_units* without setting scale does not have any effect because
176 the scale units only differ by a constant factor and that is rescaled through
177 autoscaling.
178
179units : {'width', 'height', 'dots', 'inches', 'x', 'y', 'xy'}, default: 'width'
180 Affects the arrow size (except for the length). In particular, the shaft
181 *width* is measured in multiples of this unit.
182
183 Supported values are:
184
185 - 'width', 'height': The width or height of the Axes.
186 - 'dots', 'inches': Pixels or inches based on the figure dpi.
187 - 'x', 'y', 'xy': *X*, *Y* or :math:`\\sqrt{X^2 + Y^2}` in data units.
188
189 The following table summarizes how these values affect the visible arrow
190 size under zooming and figure size changes:
191
192 ================= ================= ==================
193 units zoom figure size change
194 ================= ================= ==================
195 'x', 'y', 'xy' arrow size scales —
196 'width', 'height' — arrow size scales
197 'dots', 'inches' — —
198 ================= ================= ==================
199
200width : float, optional
201 Shaft width in arrow units. All head parameters are relative to *width*.
202
203 The default depends on choice of *units* above, and number of vectors;
204 a typical starting value is about 0.005 times the width of the plot.
205
206headwidth : float, default: 3
207 Head width as multiple of shaft *width*. See the notes below.
208
209headlength : float, default: 5
210 Head length as multiple of shaft *width*. See the notes below.
211
212headaxislength : float, default: 4.5
213 Head length at shaft intersection as multiple of shaft *width*.
214 See the notes below.
215
216minshaft : float, default: 1
217 Length below which arrow scales, in units of head length. Do not
218 set this to less than 1, or small arrows will look terrible!
219
220minlength : float, default: 1
221 Minimum length as a multiple of shaft width; if an arrow length
222 is less than this, plot a dot (hexagon) of this diameter instead.
223
224color : :mpltype:`color` or list :mpltype:`color`, optional
225 Explicit color(s) for the arrows. If *C* has been set, *color* has no
226 effect.
227
228 This is a synonym for the `.PolyCollection` *facecolor* parameter.
229
230Other Parameters
231----------------
232data : indexable object, optional
233 DATA_PARAMETER_PLACEHOLDER
234
235**kwargs : `~matplotlib.collections.PolyCollection` properties, optional
236 All other keyword arguments are passed on to `.PolyCollection`:
237
238 %(PolyCollection:kwdoc)s
239
240Returns
241-------
242`~matplotlib.quiver.Quiver`
243
244See Also
245--------
246.Axes.quiverkey : Add a key to a quiver plot.
247
248Notes
249-----
250
251**Arrow shape**
252
253The arrow is drawn as a polygon using the nodes as shown below. The values
254*headwidth*, *headlength*, and *headaxislength* are in units of *width*.
255
256.. image:: /_static/quiver_sizes.svg
257 :width: 500px
258
259The defaults give a slightly swept-back arrow. Here are some guidelines how to
260get other head shapes:
261
262- To make the head a triangle, make *headaxislength* the same as *headlength*.
263- To make the arrow more pointed, reduce *headwidth* or increase *headlength*
264 and *headaxislength*.
265- To make the head smaller relative to the shaft, scale down all the head
266 parameters proportionally.
267- To remove the head completely, set all *head* parameters to 0.
268- To get a diamond-shaped head, make *headaxislength* larger than *headlength*.
269- Warning: For *headaxislength* < (*headlength* / *headwidth*), the "headaxis"
270 nodes (i.e. the ones connecting the head with the shaft) will protrude out
271 of the head in forward direction so that the arrow head looks broken.
272""" % _docstring.interpd.params

431def _parse_args(*args, caller_name='function'):
432 """
433 Helper function to parse positional parameters for colored vector plots.
434
435 This is currently used for Quiver and Barbs.
436
437 Parameters
438 ----------
439 *args : list
440 list of 2-5 arguments. Depending on their number they are parsed to::
441
442 U, V
443 U, V, C
444 X, Y, U, V
445 X, Y, U, V, C
446
447 caller_name : str
448 Name of the calling method (used in error messages).
449 """
450 X = Y = C = None
451
452 nargs = len(args)
453 if nargs == 2:
454 # The use of atleast_1d allows for handling scalar arguments while also
455 # keeping masked arrays
456 U, V = np.atleast_1d(*args)
457 elif nargs == 3:
458 U, V, C = np.atleast_1d(*args)
459 elif nargs == 4:
460 X, Y, U, V = np.atleast_1d(*args)
461 elif nargs == 5:
462 X, Y, U, V, C = np.atleast_1d(*args)
463 else:
464 raise _api.nargs_error(caller_name, takes="from 2 to 5", given=nargs)
465
466 nr, nc = (1, U.shape[0]) if U.ndim == 1 else U.shape
467
468 if X is not None:
469 X = X.ravel()
470 Y = Y.ravel()
471 if len(X) == nc and len(Y) == nr:
472 X, Y = (a.ravel() for a in np.meshgrid(X, Y))
473 elif len(X) != len(Y):
474 raise ValueError('X and Y must be the same size, but '
475 f'X.size is {X.size} and Y.size is {Y.size}.')
476 else:
477 indexgrid = np.meshgrid(np.arange(nc), np.arange(nr))
478 X, Y = (np.ravel(a) for a in indexgrid)
479 # Size validation for U, V, C is left to the set_UVC method.
480 return X, Y, U, V, C

589 def set_UVC(self, U, V, C=None):
590 # We need to ensure we have a copy, not a reference
591 # to an array that might change before draw().
592 U = ma.masked_invalid(U, copy=True).ravel()
593 V = ma.masked_invalid(V, copy=True).ravel()
594 if C is not None:
595 C = ma.masked_invalid(C, copy=True).ravel()
596 for name, var in zip(('U', 'V', 'C'), (U, V, C)):
597 if not (var is None or var.size == self.N or var.size == 1):
598 raise ValueError(f'Argument {name} has a size {var.size}'
599 f' which does not match {self.N},'
600 ' the number of arrow positions')
601
602 mask = ma.mask_or(U.mask, V.mask, copy=False, shrink=True)
603 if C is not None:
604 mask = ma.mask_or(mask, C.mask, copy=False, shrink=True)
605 if mask is ma.nomask:
606 C = C.filled()
607 else:
608 C = ma.array(C, mask=mask, copy=False)
609 self.U = U.filled(1)
610 self.V = V.filled(1)
611 self.Umask = mask
612 if C is not None:
613 self.set_array(C)
614 self.stale = True

625 'height': bb.height,

627 'inches': self.axes.get_figure(root=True).dpi,

641 # Calculate angles and lengths for segment between (x, y), (x+u, y+v)
642 def _angles_lengths(self, XY, U, V, eps=1):
643 xy = self.axes.transData.transform(XY)
644 uv = np.column_stack((U, V))
645 xyp = self.axes.transData.transform(XY + eps * uv)
646 dxy = xyp - xy
647 angles = np.arctan2(dxy[:, 1], dxy[:, 0])
648 lengths = np.hypot(*dxy.T) / eps
649 return angles, lengths

651 # XY is stacked [X, Y].
652 # See quiver() doc for meaning of X, Y, U, V, angles.
653 def _make_verts(self, XY, U, V, angles):
654 uv = (U + V * 1j)
655 str_angles = angles if isinstance(angles, str) else ''
656 if str_angles == 'xy' and self.scale_units == 'xy':
657 # Here eps is 1 so that if we get U, V by diffing
658 # the X, Y arrays, the vectors will connect the
659 # points, regardless of the axis scaling (including log).
660 angles, lengths = self._angles_lengths(XY, U, V, eps=1)
661 elif str_angles == 'xy' or self.scale_units == 'xy':
662 # Calculate eps based on the extents of the plot
663 # so that we don't end up with roundoff error from
664 # adding a small number to a large.
665 eps = np.abs(self.axes.dataLim.extents).max() * 0.001
666 angles, lengths = self._angles_lengths(XY, U, V, eps=eps)
667
668 if str_angles and self.scale_units == 'xy':
669 a = lengths
670 else:
671 a = np.abs(uv)
672
673 if self.scale is None:
674 sn = max(10, math.sqrt(self.N))
675 if self.Umask is not ma.nomask:
676 amean = a[~self.Umask].mean()
677 else:
678 amean = a.mean()
679 # crude auto-scaling
680 # scale is typical arrow length as a multiple of the arrow width
681 scale = 1.8 * amean * sn / self.span
682
683 if self.scale_units is None:
684 if self.scale is None:
685 self.scale = scale
686 widthu_per_lenu = 1.0
687 else:
688 if self.scale_units == 'xy':
689 dx = 1
690 else:
691 dx = self._dots_per_unit(self.scale_units)
692 widthu_per_lenu = dx / self._trans_scale
693 if self.scale is None:
694 self.scale = scale * widthu_per_lenu
695 length = a * (widthu_per_lenu / (self.scale * self.width))
696 X, Y = self._h_arrows(length)
697 if str_angles == 'xy':
698 theta = angles
699 elif str_angles == 'uv':
700 theta = np.angle(uv)
701 else:
702 theta = ma.masked_invalid(np.deg2rad(angles)).filled(0)
703 theta = theta.reshape((-1, 1)) # for broadcasting
704 xy = (X + Y * 1j) * np.exp(1j * theta) * self.width
705 XY = np.stack((xy.real, xy.imag), axis=2)
706 if self.Umask is not ma.nomask:
707 XY = ma.array(XY)
708 XY[self.Umask] = ma.masked
709 # This might be handled more efficiently with nans, given
710 # that nans will end up in the paths anyway.
711
712 return XY

714 def _h_arrows(self, length):
715 """Length is in arrow width units."""
716 # It might be possible to streamline the code
717 # and speed it up a bit by using complex (x, y)
718 # instead of separate arrays; but any gain would be slight.
719 minsh = self.minshaft * self.headlength
720 N = len(length)
721 length = length.reshape(N, 1)
722 # This number is chosen based on when pixel values overflow in Agg
723 # causing rendering errors
724 # length = np.minimum(length, 2 ** 16)
725 np.clip(length, 0, 2 ** 16, out=length)
726 # x, y: normal horizontal arrow
727 x = np.array([0, -self.headaxislength,
728 -self.headlength, 0],
729 np.float64)
730 x = x + np.array([0, 1, 1, 1]) * length
731 y = 0.5 * np.array([1, 1, self.headwidth, 0], np.float64)
732 y = np.repeat(y[np.newaxis, :], N, axis=0)
733 # x0, y0: arrow without shaft, for short vectors
734 x0 = np.array([0, minsh - self.headaxislength,
735 minsh - self.headlength, minsh], np.float64)
736 y0 = 0.5 * np.array([1, 1, self.headwidth, 0], np.float64)
737 ii = [0, 1, 2, 3, 2, 1, 0, 0]
738 X = x[:, ii]
739 Y = y[:, ii]
740 Y[:, 3:-1] *= -1
741 X0 = x0[ii]
742 Y0 = y0[ii]
743 Y0[3:-1] *= -1
744 shrink = length / minsh if minsh != 0. else 0.
745 X0 = shrink * X0[np.newaxis, :]
746 Y0 = shrink * Y0[np.newaxis, :]
747 short = np.repeat(length < minsh, 8, axis=1)
748 # Now select X0, Y0 if short, otherwise X, Y
749 np.copyto(X, X0, where=short)
750 np.copyto(Y, Y0, where=short)
751 if self.pivot == 'middle':
752 X -= 0.5 * X[:, 3, np.newaxis]
753 elif self.pivot == 'tip':
754 # numpy bug? using -= does not work here unless we multiply by a
755 # float first, as with 'mid'.
756 X = X - X[:, 3, np.newaxis]
757 elif self.pivot != 'tail':
758 _api.check_in_list(["middle", "tip", "tail"], pivot=self.pivot)
759
760 tooshort = length < self.minlength
761 if tooshort.any():
762 # Use a heptagonal dot:
763 th = np.arange(0, 8, 1, np.float64) * (np.pi / 3.0)
764 x1 = np.cos(th) * self.minlength * 0.5
765 y1 = np.sin(th) * self.minlength * 0.5
766 X1 = np.repeat(x1[np.newaxis, :], N, axis=0)
767 Y1 = np.repeat(y1[np.newaxis, :], N, axis=0)
768 tooshort = np.repeat(tooshort, 8, 1)
769 np.copyto(X, X1, where=tooshort)
770 np.copyto(Y, Y1, where=tooshort)
771 # Mask handling is deferred to the caller, _make_verts.
772 return X, Y

991 def _find_tails(self, mag, rounding=True, half=5, full=10, flag=50):
992 """
993 Find how many of each of the tail pieces is necessary.
994
995 Parameters
996 ----------
997 mag : `~numpy.ndarray`
998 Vector magnitudes; must be non-negative (and an actual ndarray).
999 rounding : bool, default: True
1000 Whether to round or to truncate to the nearest half-barb.
1001 half, full, flag : float, defaults: 5, 10, 50
1002 Increments for a half-barb, a barb, and a flag.
1003
1004 Returns
1005 -------
1006 n_flags, n_barbs : int array
1007 For each entry in *mag*, the number of flags and barbs.
1008 half_flag : bool array
1009 For each entry in *mag*, whether a half-barb is needed.
1010 empty_flag : bool array
1011 For each entry in *mag*, whether nothing is drawn.
1012 """
1013 # If rounding, round to the nearest multiple of half, the smallest
1014 # increment
1015 if rounding:
1016 mag = half * np.around(mag / half)
1017 n_flags, mag = divmod(mag, flag)
1018 n_barb, mag = divmod(mag, full)
1019 half_flag = mag >= half
1020 empty_flag = ~(half_flag | (n_flags > 0) | (n_barb > 0))
1021 return n_flags.astype(int), n_barb.astype(int), half_flag, empty_flag

1023 def _make_barbs(self, u, v, nflags, nbarbs, half_barb, empty_flag, length,
1024 pivot, sizes, fill_empty, flip):
1025 """
1026 Create the wind barbs.
1027
1028 Parameters
1029 ----------
1030 u, v
1031 Components of the vector in the x and y directions, respectively.
1032
1033 nflags, nbarbs, half_barb, empty_flag
1034 Respectively, the number of flags, number of barbs, flag for
1035 half a barb, and flag for empty barb, ostensibly obtained from
1036 :meth:`_find_tails`.
1037
1038 length
1039 The length of the barb staff in points.
1040
1041 pivot : {"tip", "middle"} or number
1042 The point on the barb around which the entire barb should be
1043 rotated. If a number, the start of the barb is shifted by that
1044 many points from the origin.
1045
1046 sizes : dict
1047 Coefficients specifying the ratio of a given feature to the length
1048 of the barb. These features include:
1049
1050 - *spacing*: space between features (flags, full/half barbs).
1051 - *height*: distance from shaft of top of a flag or full barb.
1052 - *width*: width of a flag, twice the width of a full barb.
1053 - *emptybarb*: radius of the circle used for low magnitudes.
1054
1055 fill_empty : bool
1056 Whether the circle representing an empty barb should be filled or
1057 not (this changes the drawing of the polygon).
1058
1059 flip : list of bool
1060 Whether the features should be flipped to the other side of the
1061 barb (useful for winds in the southern hemisphere).
1062
1063 Returns
1064 -------
1065 list of arrays of vertices
1066 Polygon vertices for each of the wind barbs. These polygons have
1067 been rotated to properly align with the vector direction.
1068 """
1069
1070 # These control the spacing and size of barb elements relative to the
1071 # length of the shaft
1072 spacing = length * sizes.get('spacing', 0.125)
1073 full_height = length * sizes.get('height', 0.4)
1074 full_width = length * sizes.get('width', 0.25)
1075 empty_rad = length * sizes.get('emptybarb', 0.15)
1076
1077 # Controls y point where to pivot the barb.
1078 pivot_points = dict(tip=0.0, middle=-length / 2.)
1079
1080 endx = 0.0
1081 try:
1082 endy = float(pivot)
1083 except ValueError:
1084 endy = pivot_points[pivot.lower()]
1085
1086 # Get the appropriate angle for the vector components. The offset is
1087 # due to the way the barb is initially drawn, going down the y-axis.
1088 # This makes sense in a meteorological mode of thinking since there 0
1089 # degrees corresponds to north (the y-axis traditionally)
1090 angles = -(ma.arctan2(v, u) + np.pi / 2)
1091
1092 # Used for low magnitude. We just get the vertices, so if we make it
1093 # out here, it can be reused. The center set here should put the
1094 # center of the circle at the location(offset), rather than at the
1095 # same point as the barb pivot; this seems more sensible.
1096 circ = CirclePolygon((0, 0), radius=empty_rad).get_verts()
1097 if fill_empty:
1098 empty_barb = circ
1099 else:
1100 # If we don't want the empty one filled, we make a degenerate
1101 # polygon that wraps back over itself
1102 empty_barb = np.concatenate((circ, circ[::-1]))
1103
1104 barb_list = []
1105 for index, angle in np.ndenumerate(angles):
1106 # If the vector magnitude is too weak to draw anything, plot an
1107 # empty circle instead
1108 if empty_flag[index]:
1109 # We can skip the transform since the circle has no preferred
1110 # orientation
1111 barb_list.append(empty_barb)
1112 continue
1113
1114 poly_verts = [(endx, endy)]
1115 offset = length
1116
1117 # Handle if this barb should be flipped
1118 barb_height = -full_height if flip[index] else full_height
1119
1120 # Add vertices for each flag
1121 for i in range(nflags[index]):
1122 # The spacing that works for the barbs is a little to much for
1123 # the flags, but this only occurs when we have more than 1
1124 # flag.
1125 if offset != length:
1126 offset += spacing / 2.
1127 poly_verts.extend(
1128 [[endx, endy + offset],
1129 [endx + barb_height, endy - full_width / 2 + offset],
1130 [endx, endy - full_width + offset]])
1131
1132 offset -= full_width + spacing
1133
1134 # Add vertices for each barb. These really are lines, but works
1135 # great adding 3 vertices that basically pull the polygon out and
1136 # back down the line
1137 for i in range(nbarbs[index]):
1138 poly_verts.extend(
1139 [(endx, endy + offset),
1140 (endx + barb_height, endy + offset + full_width / 2),
1141 (endx, endy + offset)])
1142
1143 offset -= spacing
1144
1145 # Add the vertices for half a barb, if needed
1146 if half_barb[index]:
1147 # If the half barb is the first on the staff, traditionally it
1148 # is offset from the end to make it easy to distinguish from a
1149 # barb with a full one
1150 if offset == length:
1151 poly_verts.append((endx, endy + offset))
1152 offset -= 1.5 * spacing
1153 poly_verts.extend(
1154 [(endx, endy + offset),
1155 (endx + barb_height / 2, endy + offset + full_width / 4),
1156 (endx, endy + offset)])
1157
1158 # Rotate the barb according the angle. Making the barb first and
1159 # then rotating it made the math for drawing the barb really easy.
1160 # Also, the transform framework makes doing the rotation simple.
1161 poly_verts = transforms.Affine2D().rotate(-angle).transform(
1162 poly_verts)
1163 barb_list.append(poly_verts)
1164
1165 return barb_list
