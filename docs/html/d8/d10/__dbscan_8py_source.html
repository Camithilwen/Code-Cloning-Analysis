42 """Perform DBSCAN clustering from vector array or distance matrix.
43
44 Read more in the :ref:`User Guide <dbscan>`.
45
46 Parameters
47 ----------
48 X : {array-like, sparse (CSR) matrix} of shape (n_samples, n_features) or \
49 (n_samples, n_samples)
50 A feature array, or array of distances between samples if
51 ``metric='precomputed'``.
52
53 eps : float, default=0.5
54 The maximum distance between two samples for one to be considered
55 as in the neighborhood of the other. This is not a maximum bound
56 on the distances of points within a cluster. This is the most
57 important DBSCAN parameter to choose appropriately for your data set
58 and distance function.
59
60 min_samples : int, default=5
61 The number of samples (or total weight) in a neighborhood for a point
62 to be considered as a core point. This includes the point itself.
63
64 metric : str or callable, default='minkowski'
65 The metric to use when calculating distance between instances in a
66 feature array. If metric is a string or callable, it must be one of
67 the options allowed by :func:`sklearn.metrics.pairwise_distances` for
68 its metric parameter.
69 If metric is "precomputed", X is assumed to be a distance matrix and
70 must be square during fit.
71 X may be a :term:`sparse graph <sparse graph>`,
72 in which case only "nonzero" elements may be considered neighbors.
73
74 metric_params : dict, default=None
75 Additional keyword arguments for the metric function.
76
77 .. versionadded:: 0.19
78
79 algorithm : {'auto', 'ball_tree', 'kd_tree', 'brute'}, default='auto'
80 The algorithm to be used by the NearestNeighbors module
81 to compute pointwise distances and find nearest neighbors.
82 See NearestNeighbors module documentation for details.
83
84 leaf_size : int, default=30
85 Leaf size passed to BallTree or cKDTree. This can affect the speed
86 of the construction and query, as well as the memory required
87 to store the tree. The optimal value depends
88 on the nature of the problem.
89
90 p : float, default=2
91 The power of the Minkowski metric to be used to calculate distance
92 between points.
93
94 sample_weight : array-like of shape (n_samples,), default=None
95 Weight of each sample, such that a sample with a weight of at least
96 ``min_samples`` is by itself a core sample; a sample with negative
97 weight may inhibit its eps-neighbor from being core.
98 Note that weights are absolute, and default to 1.
99
100 n_jobs : int, default=None
101 The number of parallel jobs to run for neighbors search. ``None`` means
102 1 unless in a :obj:`joblib.parallel_backend` context. ``-1`` means
103 using all processors. See :term:`Glossary <n_jobs>` for more details.
104 If precomputed distance are used, parallel execution is not available
105 and thus n_jobs will have no effect.
106
107 Returns
108 -------
109 core_samples : ndarray of shape (n_core_samples,)
110 Indices of core samples.
111
112 labels : ndarray of shape (n_samples,)
113 Cluster labels for each point. Noisy samples are given the label -1.
114
115 See Also
116 --------
117 DBSCAN : An estimator interface for this clustering algorithm.
118 OPTICS : A similar estimator interface clustering at multiple values of
119 eps. Our implementation is optimized for memory usage.
120
121 Notes
122 -----
123 For an example, see :ref:`sphx_glr_auto_examples_cluster_plot_dbscan.py`.
124
125 This implementation bulk-computes all neighborhood queries, which increases
126 the memory complexity to O(n.d) where d is the average number of neighbors,
127 while original DBSCAN had memory complexity O(n). It may attract a higher
128 memory complexity when querying these nearest neighborhoods, depending
129 on the ``algorithm``.
130
131 One way to avoid the query complexity is to pre-compute sparse
132 neighborhoods in chunks using
133 :func:`NearestNeighbors.radius_neighbors_graph
134 <sklearn.neighbors.NearestNeighbors.radius_neighbors_graph>` with
135 ``mode='distance'``, then using ``metric='precomputed'`` here.
136
137 Another way to reduce memory and computation time is to remove
138 (near-)duplicate points and use ``sample_weight`` instead.
139
140 :class:`~sklearn.cluster.OPTICS` provides a similar clustering with lower
141 memory usage.
142
143 References

146 Algorithm for Discovering Clusters in Large Spatial Databases with Noise"
147 <https://www.dbs.ifi.lmu.de/Publikationen/Papers/KDD-96.final.frame.pdf>`_.
148 In: Proceedings of the 2nd International Conference on Knowledge Discovery

152 :doi:`"DBSCAN revisited, revisited: why and how you should (still) use DBSCAN."
153 <10.1145/3068335>`
154 ACM Transactions on Database Systems (TODS), 42(3), 19.
155
156 Examples
157 --------
158 >>> from sklearn.cluster import dbscan
159 >>> X = [[1, 2], [2, 2], [2, 3], [8, 7], [8, 8], [25, 80]]
160 >>> core_samples, labels = dbscan(X, eps=3, min_samples=2)
161 >>> core_samples
162 array([0, 1, 2, 3, 4])
163 >>> labels
164 array([ 0, 0, 0, 1, 1, -1])

181class DBSCAN(ClusterMixin, BaseEstimator):
182 """Perform DBSCAN clustering from vector array or distance matrix.
183
184 DBSCAN - Density-Based Spatial Clustering of Applications with Noise.
185 Finds core samples of high density and expands clusters from them.
186 Good for data which contains clusters of similar density.
187
188 This implementation has a worst case memory complexity of :math:`O({n}^2)`,
189 which can occur when the `eps` param is large and `min_samples` is low,
190 while the original DBSCAN only uses linear memory.
191 For further details, see the Notes below.
192
193 Read more in the :ref:`User Guide <dbscan>`.
194
195 Parameters
196 ----------
197 eps : float, default=0.5
198 The maximum distance between two samples for one to be considered
199 as in the neighborhood of the other. This is not a maximum bound
200 on the distances of points within a cluster. This is the most
201 important DBSCAN parameter to choose appropriately for your data set
202 and distance function.
203
204 min_samples : int, default=5
205 The number of samples (or total weight) in a neighborhood for a point to
206 be considered as a core point. This includes the point itself. If
207 `min_samples` is set to a higher value, DBSCAN will find denser clusters,
208 whereas if it is set to a lower value, the found clusters will be more
209 sparse.
210
211 metric : str, or callable, default='euclidean'
212 The metric to use when calculating distance between instances in a
213 feature array. If metric is a string or callable, it must be one of
214 the options allowed by :func:`sklearn.metrics.pairwise_distances` for
215 its metric parameter.
216 If metric is "precomputed", X is assumed to be a distance matrix and
217 must be square. X may be a :term:`sparse graph`, in which
218 case only "nonzero" elements may be considered neighbors for DBSCAN.
219
220 .. versionadded:: 0.17
221 metric *precomputed* to accept precomputed sparse matrix.
222
223 metric_params : dict, default=None
224 Additional keyword arguments for the metric function.
225
226 .. versionadded:: 0.19
227
228 algorithm : {'auto', 'ball_tree', 'kd_tree', 'brute'}, default='auto'
229 The algorithm to be used by the NearestNeighbors module
230 to compute pointwise distances and find nearest neighbors.
231 See NearestNeighbors module documentation for details.
232
233 leaf_size : int, default=30
234 Leaf size passed to BallTree or cKDTree. This can affect the speed
235 of the construction and query, as well as the memory required
236 to store the tree. The optimal value depends
237 on the nature of the problem.
238
239 p : float, default=None
240 The power of the Minkowski metric to be used to calculate distance
241 between points. If None, then ``p=2`` (equivalent to the Euclidean
242 distance).
243
244 n_jobs : int, default=None
245 The number of parallel jobs to run.
246 ``None`` means 1 unless in a :obj:`joblib.parallel_backend` context.
247 ``-1`` means using all processors. See :term:`Glossary <n_jobs>`
248 for more details.
249
250 Attributes
251 ----------
252 core_sample_indices_ : ndarray of shape (n_core_samples,)
253 Indices of core samples.
254
255 components_ : ndarray of shape (n_core_samples, n_features)
256 Copy of each core sample found by training.
257
258 labels_ : ndarray of shape (n_samples)
259 Cluster labels for each point in the dataset given to fit().
260 Noisy samples are given the label -1.
261
262 n_features_in_ : int
263 Number of features seen during :term:`fit`.
264
265 .. versionadded:: 0.24
266
267 feature_names_in_ : ndarray of shape (`n_features_in_`,)
268 Names of features seen during :term:`fit`. Defined only when `X`
269 has feature names that are all strings.
270
271 .. versionadded:: 1.0
272
273 See Also
274 --------
275 OPTICS : A similar clustering at multiple values of eps. Our implementation
276 is optimized for memory usage.
277

366 # DBSCAN.metric is not validated yet
367 prefer_skip_nested_validation=False
368 )
369 def fit(self, X, y=None, sample_weight=None):
370 """Perform DBSCAN clustering from features, or distance matrix.
371
372 Parameters
373 ----------
374 X : {array-like, sparse matrix} of shape (n_samples, n_features), or \
375 (n_samples, n_samples)
376 Training instances to cluster, or distances between instances if
377 ``metric='precomputed'``. If a sparse matrix is provided, it will
378 be converted into a sparse ``csr_matrix``.
379
380 y : Ignored
381 Not used, present here for API consistency by convention.
382
383 sample_weight : array-like of shape (n_samples,), default=None
384 Weight of each sample, such that a sample with a weight of at least
385 ``min_samples`` is by itself a core sample; a sample with a
386 negative weight may inhibit its eps-neighbor from being core.
387 Note that weights are absolute, and default to 1.
388
389 Returns
390 -------
391 self : object
392 Returns a fitted instance of self.
393 """
394 X = validate_data(self, X, accept_sparse="csr")
395
396 if sample_weight is not None:
397 sample_weight = _check_sample_weight(sample_weight, X)
398
399 # Calculate neighborhood for all samples. This leaves the original
400 # point in, which needs to be considered later (i.e. point i is in the
401 # neighborhood of point i. While True, its useless information)
402 if self.metric == "precomputed" and sparse.issparse(X):
403 # set the diagonal to explicit values, as a point is its own
404 # neighbor
405 X = X.copy() # copy to avoid in-place modification
406 with warnings.catch_warnings():
407 warnings.simplefilter("ignore", sparse.SparseEfficiencyWarning)
408 X.setdiag(X.diagonal())
409
410 neighbors_model = NearestNeighbors(
411 radius=self.eps,
412 algorithm=self.algorithm,
413 leaf_size=self.leaf_size,
414 metric=self.metric,
415 metric_params=self.metric_params,
416 p=self.p,
417 n_jobs=self.n_jobs,
418 )
419 neighbors_model.fit(X)
420 # This has worst case O(n^2) memory complexity
421 neighborhoods = neighbors_model.radius_neighbors(X, return_distance=False)
422
423 if sample_weight is None:
424 n_neighbors = np.array([len(neighbors) for neighbors in neighborhoods])
425 else:
426 n_neighbors = np.array(
427 [np.sum(sample_weight[neighbors]) for neighbors in neighborhoods]
428 )
429
430 # Initially, all samples are noise.
431 labels = np.full(X.shape[0], -1, dtype=np.intp)
432
433 # A list of all core samples found.
434 core_samples = np.asarray(n_neighbors >= self.min_samples, dtype=np.uint8)
435 dbscan_inner(core_samples, neighborhoods, labels)
436
437 self.core_sample_indices_ = np.where(core_samples)[0]
438 self.labels_ = labels
439
440 if len(self.core_sample_indices_):
441 # fix for scipy sparse indexing issue
442 self.components_ = X[self.core_sample_indices_].copy()
443 else:
444 # no core samples
445 self.components_ = np.empty((0, X.shape[1]))
446 return self
