155def _check_for_locals(expr: str, stack_level: int, parser: str):
156 at_top_of_stack = stack_level == 0
157 not_pandas_parser = parser != "pandas"
158
159 if not_pandas_parser:
160 msg = "The '@' prefix is only supported by the pandas parser"
161 elif at_top_of_stack:
162 msg = (
163 "The '@' prefix is not allowed in top-level eval calls.\n"
164 "please refer to your variables by name without the '@' prefix."
165 )
166
167 if at_top_of_stack or not_pandas_parser:
168 for toknum, tokval in tokenize_string(expr):
169 if toknum == tokenize.OP and tokval == "@":
170 raise SyntaxError(msg)

173def eval(
174 expr: str | BinOp, # we leave BinOp out of the docstr bc it isn't for users
175 parser: str = "pandas",
176 engine: str | None = None,
177 local_dict=None,
178 global_dict=None,
179 resolvers=(),
180 level: int = 0,
181 target=None,
182 inplace: bool = False,
183):
184 """
185 Evaluate a Python expression as a string using various backends.
186
187 The following arithmetic operations are supported: ``+``, ``-``, ``*``,
188 ``/``, ``**``, ``%``, ``//`` (python engine only) along with the following
189 boolean operations: ``|`` (or), ``&`` (and), and ``~`` (not).
190 Additionally, the ``'pandas'`` parser allows the use of :keyword:`and`,
191 :keyword:`or`, and :keyword:`not` with the same semantics as the
192 corresponding bitwise operators. :class:`~pandas.Series` and
193 :class:`~pandas.DataFrame` objects are supported and behave as they would
194 with plain ol' Python evaluation.
195
196 Parameters
197 ----------
198 expr : str
199 The expression to evaluate. This string cannot contain any Python
200 `statements
201 <https://docs.python.org/3/reference/simple_stmts.html#simple-statements>`__,
202 only Python `expressions
203 <https://docs.python.org/3/reference/simple_stmts.html#expression-statements>`__.
204 parser : {'pandas', 'python'}, default 'pandas'
205 The parser to use to construct the syntax tree from the expression. The
206 default of ``'pandas'`` parses code slightly different than standard
207 Python. Alternatively, you can parse an expression using the
208 ``'python'`` parser to retain strict Python semantics. See the
209 :ref:`enhancing performance <enhancingperf.eval>` documentation for
210 more details.
211 engine : {'python', 'numexpr'}, default 'numexpr'
212
213 The engine used to evaluate the expression. Supported engines are
214
215 - None : tries to use ``numexpr``, falls back to ``python``
216 - ``'numexpr'`` : This default engine evaluates pandas objects using
217 numexpr for large speed ups in complex expressions with large frames.
218 - ``'python'`` : Performs operations as if you had ``eval``'d in top
219 level python. This engine is generally not that useful.
220
221 More backends may be available in the future.
222 local_dict : dict or None, optional
223 A dictionary of local variables, taken from locals() by default.
224 global_dict : dict or None, optional
225 A dictionary of global variables, taken from globals() by default.
226 resolvers : list of dict-like or None, optional
227 A list of objects implementing the ``__getitem__`` special method that
228 you can use to inject an additional collection of namespaces to use for
229 variable lookup. For example, this is used in the
230 :meth:`~DataFrame.query` method to inject the
231 ``DataFrame.index`` and ``DataFrame.columns``
232 variables that refer to their respective :class:`~pandas.DataFrame`
233 instance attributes.
234 level : int, optional
235 The number of prior stack frames to traverse and add to the current
236 scope. Most users will **not** need to change this parameter.
237 target : object, optional, default None
238 This is the target object for assignment. It is used when there is
239 variable assignment in the expression. If so, then `target` must
240 support item assignment with string keys, and if a copy is being
241 returned, it must also support `.copy()`.
242 inplace : bool, default False
243 If `target` is provided, and the expression mutates `target`, whether
244 to modify `target` inplace. Otherwise, return a copy of `target` with
245 the mutation.
246
247 Returns
248 -------
249 ndarray, numeric scalar, DataFrame, Series, or None
250 The completion value of evaluating the given code or None if ``inplace=True``.
251
252 Raises
253 ------
254 ValueError
255 There are many instances where such an error can be raised:
256
257 - `target=None`, but the expression is multiline.
258 - The expression is multiline, but not all them have item assignment.
259 An example of such an arrangement is this:
260
261 a = b + 1
262 a + 2
263
264 Here, there are expressions on different lines, making it multiline,
265 but the last line has no variable assigned to the output of `a + 2`.
266 - `inplace=True`, but the expression is missing item assignment.
267 - Item assignment is provided, but the `target` does not support
268 string item assignment.
269 - Item assignment is provided and `inplace=False`, but the `target`
270 does not support the `.copy()` method
271
272 See Also
273 --------
274 DataFrame.query : Evaluates a boolean expression to query the columns
275 of a frame.
276 DataFrame.eval : Evaluate a string describing operations on
277 DataFrame columns.
278
279 Notes
280 -----
281 The ``dtype`` of any objects involved in an arithmetic ``%`` operation are
282 recursively cast to ``float64``.
283
284 See the :ref:`enhancing performance <enhancingperf.eval>` documentation for
285 more details.
286
287 Examples
288 --------
289 >>> df = pd.DataFrame({"animal": ["dog", "pig"], "age": [10, 20]})
290 >>> df
291 animal age
292 0 dog 10
293 1 pig 20
294
295 We can add a new column using ``pd.eval``:
296
297 >>> pd.eval("double_age = df.age * 2", target=df)
298 animal age double_age
299 0 dog 10 20
300 1 pig 20 40
