258 """
259 Create an inset axes with a given width and height.
260
261 Both sizes used can be specified either in inches or percentage.
262 For example,::
263
264 inset_axes(parent_axes, width='40%%', height='30%%', loc='lower left')
265
266 creates in inset axes in the lower left corner of *parent_axes* which spans
267 over 30%% in height and 40%% in width of the *parent_axes*. Since the usage
268 of `.inset_axes` may become slightly tricky when exceeding such standard
269 cases, it is recommended to read :doc:`the examples
270 </gallery/axes_grid1/inset_locator_demo>`.
271
272 Notes
273 -----
274 The meaning of *bbox_to_anchor* and *bbox_to_transform* is interpreted
275 differently from that of legend. The value of bbox_to_anchor
276 (or the return value of its get_points method; the default is
277 *parent_axes.bbox*) is transformed by the bbox_transform (the default
278 is Identity transform) and then interpreted as points in the pixel
279 coordinate (which is dpi dependent).
280
281 Thus, following three calls are identical and creates an inset axes
282 with respect to the *parent_axes*::
283
284 axins = inset_axes(parent_axes, "30%%", "40%%")
285 axins = inset_axes(parent_axes, "30%%", "40%%",
286 bbox_to_anchor=parent_axes.bbox)
287 axins = inset_axes(parent_axes, "30%%", "40%%",
288 bbox_to_anchor=(0, 0, 1, 1),
289 bbox_transform=parent_axes.transAxes)
290
291 Parameters
292 ----------
293 parent_axes : `matplotlib.axes.Axes`
294 Axes to place the inset axes.
295
296 width, height : float or str
297 Size of the inset axes to create. If a float is provided, it is
298 the size in inches, e.g. *width=1.3*. If a string is provided, it is
299 the size in relative units, e.g. *width='40%%'*. By default, i.e. if
300 neither *bbox_to_anchor* nor *bbox_transform* are specified, those
301 are relative to the parent_axes. Otherwise, they are to be understood
302 relative to the bounding box provided via *bbox_to_anchor*.
303
304 loc : str, default: 'upper right'
305 Location to place the inset axes. Valid locations are
306 'upper left', 'upper center', 'upper right',
307 'center left', 'center', 'center right',
308 'lower left', 'lower center', 'lower right'.
309 For backward compatibility, numeric values are accepted as well.
310 See the parameter *loc* of `.Legend` for details.
311
312 bbox_to_anchor : tuple or `~matplotlib.transforms.BboxBase`, optional
313 Bbox that the inset axes will be anchored to. If None,
314 a tuple of (0, 0, 1, 1) is used if *bbox_transform* is set
315 to *parent_axes.transAxes* or *parent_axes.figure.transFigure*.
316 Otherwise, *parent_axes.bbox* is used. If a tuple, can be either
317 [left, bottom, width, height], or [left, bottom].
318 If the kwargs *width* and/or *height* are specified in relative units,
319 the 2-tuple [left, bottom] cannot be used. Note that,
320 unless *bbox_transform* is set, the units of the bounding box
321 are interpreted in the pixel coordinate. When using *bbox_to_anchor*
322 with tuple, it almost always makes sense to also specify
323 a *bbox_transform*. This might often be the axes transform
324 *parent_axes.transAxes*.
325
326 bbox_transform : `~matplotlib.transforms.Transform`, optional
327 Transformation for the bbox that contains the inset axes.
328 If None, a `.transforms.IdentityTransform` is used. The value
329 of *bbox_to_anchor* (or the return value of its get_points method)
330 is transformed by the *bbox_transform* and then interpreted
331 as points in the pixel coordinate (which is dpi dependent).
332 You may provide *bbox_to_anchor* in some normalized coordinate,
333 and give an appropriate transform (e.g., *parent_axes.transAxes*).
334
335 axes_class : `~matplotlib.axes.Axes` type, default: `.HostAxes`
336 The type of the newly created inset axes.
337
338 axes_kwargs : dict, optional
339 Keyword arguments to pass to the constructor of the inset axes.
340 Valid arguments include:
341
342 %(Axes:kwdoc)s
343
344 borderpad : float, default: 0.5
345 Padding between inset axes and the bbox_to_anchor.
346 The units are axes font size, i.e. for a default font size of 10 points
347 *borderpad = 0.5* is equivalent to a padding of 5 points.
348
349 Returns
350 -------
351 inset_axes : *axes_class*
352 Inset axes object created.
353 """
354
355 if (bbox_transform in [parent_axes.transAxes,
356 parent_axes.get_figure(root=False).transFigure]
357 and bbox_to_anchor is None):
358 _api.warn_external("Using the axes or figure transform requires a "
359 "bounding box in the respective coordinates. "
360 "Using bbox_to_anchor=(0, 0, 1, 1) now.")
361 bbox_to_anchor = (0, 0, 1, 1)
362 if bbox_to_anchor is None:
363 bbox_to_anchor = parent_axes.bbox
364 if (isinstance(bbox_to_anchor, tuple) and
365 (isinstance(width, str) or isinstance(height, str))):
366 if len(bbox_to_anchor) != 4:
367 raise ValueError("Using relative units for width or height "
368 "requires to provide a 4-tuple or a "
369 "`Bbox` instance to `bbox_to_anchor.")
370 return _add_inset_axes(
371 parent_axes, axes_class, axes_kwargs,
372 AnchoredSizeLocator(
373 bbox_to_anchor, width, height, loc=loc,
374 bbox_transform=bbox_transform, borderpad=borderpad))
