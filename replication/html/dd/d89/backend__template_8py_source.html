1"""
2A fully functional, do-nothing backend intended as a template for backend
3writers. It is fully functional in that you can select it as a backend e.g.
4with ::
5
6 import matplotlib
7 matplotlib.use("template")
8
9and your program will (should!) run without error, though no output is
10produced. This provides a starting point for backend writers; you can
11selectively implement drawing methods (`~.RendererTemplate.draw_path`,
12`~.RendererTemplate.draw_image`, etc.) and slowly see your figure come to life
13instead having to have a full-blown implementation before getting any results.
14
15Copy this file to a directory outside the Matplotlib source tree, somewhere
16where Python can import it (by adding the directory to your ``sys.path`` or by
17packaging it as a normal Python package); if the backend is importable as
18``import my.backend`` you can then select it using ::
19
20 import matplotlib
21 matplotlib.use("module://my.backend")
22
23If your backend implements support for saving figures (i.e. has a ``print_xyz`` method),
24you can register it as the default handler for a given file type::
25
26 from matplotlib.backend_bases import register_backend
27 register_backend('xyz', 'my_backend', 'XYZ File Format')
28 ...
29 plt.savefig("figure.xyz")
30"""

109class GraphicsContextTemplate(GraphicsContextBase):
110 """
111 The graphics context provides the color, line styles, etc. See the cairo
112 and postscript backends for examples of mapping the graphics context
113 attributes (cap styles, join styles, line widths, colors) to a particular
114 backend. In cairo this is done by wrapping a cairo.Context object and
115 forwarding the appropriate calls to it using a dictionary mapping styles
116 to gdk constants. In Postscript, all the work is done by the renderer,
117 mapping line styles to postscript calls.
118
119 If it's more appropriate to do the mapping at the renderer level (as in
120 the postscript backend), you don't need to override any of the GC methods.
121 If it's more appropriate to wrap an instance (as in the cairo backend) and
122 do the mapping here, you'll need to override several of the setter
123 methods.
124
125 The base GraphicsContext stores colors as an RGB tuple on the unit
126 interval, e.g., (0.5, 0.0, 1.0). You may need to map this to colors
127 appropriate for your backend.
128 """
